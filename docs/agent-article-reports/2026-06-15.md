## Search Agent

### 1. CoRe: A Continuously Reward-Finetuned LLM Query Rewriter for Multi-Stage Context-Aware Relevance in Web-Scale Video Search

- 链接：[arXiv](https://arxiv.org/abs/2606.14127) | [PDF](https://arxiv.org/pdf/2606.14127)
- 证据来源：PDF 正文抽取
- 问题背景：生产搜索里的 LLM query rewriter 不是单纯“把 query 写得更漂亮”。它的 rewrite 会被召回、粗排、精排共同消费，所以训练奖励如果只用离线检索指标，容易和线上排序融合逻辑错位；同时线上分布每周漂移，训练还必须足够便宜，才能持续重训和自动上线。
- 核心方法：CoRe 把 deployed multimodal relevance model 直接作为奖励来源，并把奖励形式做成与生产 fusion algebra 相似的乘法比例，减少离线代理指标和线上 ranker 的差距。训练上使用 semi-online Mixed Preference Optimization：每个 phase 先采样 rewrite，再只对 top-k / bottom-k 轨迹构造偏好对，避免每个梯度步都同步 trainer 和 inference server。上线前还有三类 reward-like 指标和四类 stability 指标组成的自动 promotion gate。
- 创新点：它真正把“搜索 agent 的动作奖励”接到生产 ranker 上，而不是用通用 relevance proxy。更重要的是，论文报告自动 gate 曾发现并拦截真实 reward-hacking：rewrite 变长带来的奖励表面提升没有骗过 stability 指标。这对 Agent RL 很关键，因为工具使用轨迹的奖励也很容易被 verbosity、无效调用或格式 hack 污染。
- 实验与 benchmark：论文记录了 5 个多月、20 次 weekly gate 的生产自动 promotion 窗口；4 次候选被 hold，其中 3 次仅由 stability metrics 捕获。线上 A/B 分两阶段：先在 finerank 使用 rewriter，再扩展到 recall + rawrank。两阶段都报告 rewrite-impacted queries 的 change-query rate 显著下降，主要 relevance 和 engagement 指标方向一致。
- 可转化价值：适合直接借鉴到 Search Agent 后训练：用下游系统真实消费方式定义奖励，用稳定性指标约束轨迹形态，用 weekly semi-online loop 控制成本，并保留原始信号作为并行路径降低失败半径。
- 局限：证据来自单一大型短视频搜索系统，奖励模型和线上融合逻辑强绑定；迁移到开放 Web Search Agent 时需要重新定义 downstream reward source。
- 阅读建议：今天第一优先级，尤其值得关注 reward shape、promotion gate 和并行路径部署。

### 4. ScoreGate: Adaptive Chunk Selection for Retrieval-Augmented Generation via Dual-Score Statistical Fusion

- 链接：[arXiv](https://arxiv.org/abs/2606.14269) | [PDF](https://arxiv.org/pdf/2606.14269)
- 证据来源：PDF 正文抽取
- 问题背景：RAG 常用固定 Top-K，把所有 query 都塞同样数量 chunk。窄问题会过检索，组合问题会欠检索；更麻烦的是，bi-encoder 因词汇不匹配错排的相关 chunk，固定阈值或单一 reranker 阈值不一定能救回来。
- 核心方法：ScoreGate 不额外调用模型，而是融合标准检索流水线已有的两个分数：bi-encoder similarity 和 cross-encoder reranker score。它把候选分成若干 score bucket，并用不对称阈值保留“embedding 排名不高但 cross-encoder 强确认”的 chunk，从而动态决定送入 generator 的 chunk 数量。
- 创新点：它处理的是 retrieval cardinality，而不是 reranking 或 retrieve/no-retrieve 触发。对 Search Agent 更像一个低成本动作门控器：query 难度和证据质量不同，agent 不应固定阅读同样多上下文。
- 实验与 benchmark：公共 MS MARCO 200 dev queries 上，ScoreGate 报告 MRR@10=0.401，同时比 Standard Top-K 少保留 35% chunk。内部 ARB 300 条标注样本中，观察到 0 false positives，95% CI 为 [96.4%, 100%]，召回 97.77%-99.34%，tokens/query 减少 34.8%，延迟增加 31ms。论文明确说明内部 ARB 是 tuned domain validation，MS MARCO 才是较干净的外部测试。
- 可转化价值：可作为 Search Agent 的 evidence budget controller：在检索轨迹里记录双分数、bucket、保留/丢弃动作，把“读多少证据”变成可训练或可审计策略。
- 局限：内部集和调参知识库共享，泛化证据有限；对 dense retrieval + cross-encoder 架构以外的系统需要重调阈值。
- 阅读建议：适合做 RAG 工具调用成本控制和检索证据选择模块。

### 5. Verifiable User Simulation for Search and Recommendation Systems

- 链接：[arXiv](https://arxiv.org/abs/2606.14474) | [PDF](https://arxiv.org/pdf/2606.14474)
- 证据来源：PDF 正文抽取
- 问题背景：LLM user simulator 被用来评估搜索、推荐和 RAG，但很多 simulator 是黑箱：不知道选择来自 persona、task contract、模型偏差，还是 prompt 偶然性。对少数群体、公平性和长交互评估尤其危险。
- 核心方法：论文是半天 tutorial，提出七个可审计组件：Persona、Contract、human-vs-agent Execution、Trace、persona-aligned Verification、Feedback、Refinement。mini-lab 覆盖推荐列表评估和 search query formulation，让参与者比较人类和 agent trace，并把差异归因到 persona 模糊、contract 不完整、agent 偏离、人类扮演差异或随机性。
- 创新点：它把 user simulator 当作工程 artefact，而不是 prompt persona。对 Search Agent 评测很有启发：模拟用户的每一步都要有 trace、verification rubric 和 refinement loop，否则 simulator 只能制造看似真实的交互数据，不能证明其 fidelity。
- 实验与 benchmark：这是 tutorial / framework paper，不是传统 benchmark 论文。正文给出 180 分钟 session 设计、85 分钟 hands-on lab、Persona/Contract JSON schema、Trace schema、verification rubric 和反馈表，但没有报告模型间定量指标。
- 可转化价值：适合用来设计 Search Agent / Browser Agent 的用户模拟数据管线：persona 和任务约束结构化，执行轨迹可回放，失败归因可进入下一轮 persona/contract 修订。
- 局限：缺少定量实验；推荐系统部分对当前 Agent 研究是相邻领域，需要避免把推荐任务本身误当作 Search Agent 核心贡献。
- 阅读建议：作为评测和数据构造方法读，不作为算法 SOTA 读。

### 6. Knowledge Graph Enhanced Memory-Augmented Retrieval for Long Context Modeling

- 链接：[arXiv](https://arxiv.org/abs/2606.14047) | [PDF](https://arxiv.org/pdf/2606.14047)
- 证据来源：PDF 正文抽取
- 问题背景：长上下文模型只扩大窗口不够，agent 在长任务里还需要维护实体状态和关系。纯语义相似度检索容易找回表面相近文本，却丢失跨段实体关系。
- 核心方法：KGERMAR 在推理时从输入文本动态构建 context-specific knowledge graph，做实体和关系抽取，再把 graph structural embedding 与文本语义一起送入多组件 memory architecture。系统维护 contextual、semantic、structural 三类 memory bank，并用 learned weights 融合检索信号。
- 创新点：它不是依赖 ConceptNet / Freebase 这类固定知识图谱，而是每个输入上下文即时生成图结构。对 Agent memory 来说，这相当于把“任务内实体关系”变成可检索状态，而不只是一堆 chunk。
- 实验与 benchmark：Fine-tune 用 SlimPajama 84.7K examples；评估包括 WikiText-103 4,358 examples、PG-19 100 examples、Proof-pile 46.3K examples，以及 SST-2、MR、Subj、SST-5、MPQA 的 few-shot NLU。论文报告 perplexity 最多降低 8.5%，1K-32K context 下 memory efficiency 提升 2-2.5x，32K 时显存使用优势更明显。
- 可转化价值：适合迁移到长程工具任务的 memory compression：从日志和网页中抽取实体-关系图，再把结构 memory 与语义 chunk 一起检索。
- 局限：评测仍偏 language modeling / NLU，不是端到端 Agent benchmark；实时图构建的错误传播和成本需要进一步验证。
- 阅读建议：作为 long-context Agent memory 组件参考，优先级低于 CoRe 和 ScoreGate。

## Code Agent

### 2. VHDLSuite: Unified Pipeline for LLM VHDL Generation with Data Synthesis and Evaluation

- 链接：[arXiv](https://arxiv.org/abs/2606.13735) | [PDF](https://arxiv.org/pdf/2606.13735)
- 证据来源：PDF 正文抽取
- 问题背景：LLM RTL/HDL generation 主要被 Verilog benchmark 覆盖，VHDL 的强类型、严格语义、并发/时序表达和 testbench 组织方式不同，直接用 Verilog 成绩不能说明模型能否泛化到 VHDL。
- 核心方法：VHDLSuite 把成熟 Verilog benchmark 转换成可执行 VHDL benchmark。构造阶段使用 LLM translation、语法修复和 simulation-driven iterative validation，只有能在 VUnit/GHDL harness 下编译并通过仿真的任务才进入 VHDLBench。评测阶段则严格 no-feedback：模型只拿题目，不拿 testbench、Verilog reference 或仿真反馈，输出后统一由 VUnit/GHDL 检查。
- 创新点：它把 benchmark 合成和 benchmark 评测分开，用可执行验证保证数据质量。对 Code Agent 很重要，因为“生成代码 + 运行测试 + 诊断失败类型”比只看文本相似度更接近真实工具闭环。
- 实验与 benchmark：VHDLBench 含 206 个问题，来源包括 VerilogEval v2 和 RTLLM v2。论文评估 Claude Sonnet 4.5、DeepSeek V3.2 Speciale、Gemini 3 Pro Preview、GLM 4.6V、GPT 5.1 Codex Max、Grok 4、Qwen3 Max，报告 pass@1/3/5、编译/仿真错误、AST syntactic similarity、失败模式分类等。正文指出 Gemini 3 Pro Preview pass@k 领先，GPT 5.1 Codex Max 和 Grok 4 pass@5 接近同组但 pass@1 较低，DeepSeek V3.2 Speciale 在结构/语法错误上有独特表现。
- 可转化价值：适合用于 Code Agent 数据构造：先用工具链生成可执行任务，再用统一 verifier 评测多模型，并把编译错误、类型错误、仿真失败拆成可训练反馈。
- 局限：VHDL 任务边界清晰、仿真器可靠；普通软件仓库需要处理依赖、外部服务和 flaky tests。
- 阅读建议：今天 Code Agent 第一优先级。

### 7. Quasilinear Equivalence Checking for Detector Error Models

- 链接：[arXiv](https://arxiv.org/abs/2606.14677) | [PDF](https://arxiv.org/pdf/2606.14677)
- 证据来源：PDF 正文抽取
- 问题背景：量子纠错编译管线里，Detector Error Model 描述错误机制、detectors 和 observables。要验证编译/优化是否保持 fault-tolerance，需要判断两个 DEM 是否 decoder-equivalent；传统做法可能依赖采样或展开 repeat block，效率和保证都不足。
- 核心方法：论文为 DEM 建立 term language、scope-local equivalence 和重写系统，证明系统 sound、terminating、confluent，因此每个 term 有唯一 normal form。对 flat DEM，normal-form Tanner graph 加概率赋值成为完整不变量；比较 normal form 即可做静态等价检查，复杂度为 O(k|E|log|E|)。
- 创新点：它不是 LLM 论文，但对 Code Agent 的 verification 思路有迁移意义：当领域语言可正规化时，应该优先把“代码是否等价”转成规范形比较，而不是只靠测试采样。
- 实验与 benchmark：论文主要是理论和算法正确性证明，没有常规 LLM benchmark。核心结果是 normal form theorem、complete invariants、quasilinear decision procedure，并讨论非自适应 QEC pipeline 中 scope-local equivalence 与 full decoder-equivalence 的关系。
- 可转化价值：适合放在 code verification / compiler agent 的 related work：把 DSL、配置、查询计划等输出转成 canonical form，用静态等价器给 Agent 轨迹奖励。
- 局限：领域非常窄，直接 Agent 训练价值低；迁移需要目标语言具备可形式化语义。
- 阅读建议：作为“形式化 verifier 优先于采样测试”的方法论参考。

### 3. MeEvo: Metacognitive Evolution Combined with Natural Evolution for Automatic Heuristic Design

- 链接：[arXiv](https://arxiv.org/abs/2606.14202) | [PDF](https://arxiv.org/pdf/2606.14202)
- 证据来源：PDF 正文抽取
- 问题背景：LLM automatic heuristic design 已有两类路线：Natural Evolution 用 crossover/mutation 搜索 heuristic program，但通常丢掉 reasoning trace；Metacognitive Evolution 用反思改进 reasoning，却缺少 population-level recombination，容易局部收敛。
- 核心方法：MeEvo 把两层循环起来。Natural Evolution 负责探索 heuristic code，同时把 reasoning traces、fitness values 和 errors 写入 shared history；Metacognitive Evolution 读取历史，提炼 Meta Insight，再生成改进 heuristic 重新进入 parent pool。直观上，它把“代码种群”和“可继承的思考记录”分开保存，再周期性交换信息。
- 创新点：它把反思从辅助 prompt 变成独立 evolutionary layer，使 reasoning trace 变成可遗传信息。对 General Agent 后训练有启发：工具执行错误、fitness、修复理由都应该进入 shared history，而不是每轮生成完就丢。
- 实验与 benchmark：论文在五个优化问题上评估：TSP-Construct、TSP-ACO、BPP-ACO、ACS、WSN，并使用 DeepSeek-V4-Flash 和 MIMO-v2.5-Pro 两个 LLM backbone。正文报告 MeEvo 在两个 backbone 上取得最优平均表现，TSP-ACO 最优值 5.712、BPP-ACO 204.20，并在复杂约束问题上更稳定；消融显示只保留 Natural Evolution 或只保留 Metacognitive Evolution 都不如双层交替。
- 可转化价值：适合用于 Agent 训练数据组织：把代码/策略轨迹、错误、fitness 和反思摘要都作为可复用 memory，形成“探索-反思-再探索”的后训练样本。
- 局限：任务是优化启发式设计，不是通用工具使用；结果依赖 prompt、population size 和问题可快速评估。
- 阅读建议：适合关注 self-improvement、reflection memory 和 evolutionary search 的读者。

### 8. Operator Calculus for Population-Based Optimization: A Mean-Field Convergence Theory

- 链接：[arXiv](https://arxiv.org/abs/2606.14289) | [PDF](https://arxiv.org/pdf/2606.14289)
- 证据来源：PDF 正文抽取
- 问题背景：进化策略、CMA、consensus-based optimization、distributional SGD 等 population-based 方法都有各自收敛分析，难以复用到组合算法。
- 核心方法：论文提出 operator calculus，把此类算法表示为 mutation、selection、recombination 三类 elementary operators 在概率测度上的组合；在稳定性和正则条件下，组合 operator 有 transport-reaction-jump PDE 连续时间极限，并可用 modular Lyapunov principle 组合证明收敛。
- 创新点：对 Agent 直接贡献不强，但可作为 population-based agent training 或多策略搜索的理论工具。特别是 MeEvo 这类 LLM heuristic evolution 如果要分析稳定性，可以借鉴 operator-level 分解。
- 实验与 benchmark：主要是理论论文，没有应用 benchmark；贡献集中在算子分解、pre-generator、TRJ PDE 和 Lyapunov 收敛条件。
- 可转化价值：适合用于研究 population RL / agentic search 的理论 related work。
- 局限：工程可操作性低，短期不建议投入实现。
- 阅读建议：低优先级，除非正在写 evolution-based agent training 理论部分。

### 9. ChronoID: Infusing Explicit Temporal Signals into Semantic IDs for Generative Recommendation

- 链接：[arXiv](https://arxiv.org/abs/2606.14260) | [PDF](https://arxiv.org/pdf/2606.14260)
- 证据来源：PDF 正文抽取
- 问题背景：Generative recommendation 常把 item 表示离散化成 semantic IDs，但 ID 学习通常不显式编码时间，导致不同时间语境下的交互被压成同一 token。
- 核心方法：ChronoID 系统化研究时间信号如何进入 semantic ID：绝对时间 vs 相对间隔、时间和 item embedding 的融合顺序、parallel/residual quantization。它还构造 time-explicit benchmark，严格按时间切分，避免未来信息泄漏。
- 创新点：和 Agent 的关系较远，但“把时间上下文编进离散 token”对长期 memory indexing 有一点参考价值：同一工具/网页/文件在不同任务阶段可能应有不同状态 ID。
- 实验与 benchmark：论文基于 Mercari、Amazon Industrial and Office 等推荐数据构造 time-explicit generative recommendation benchmark，并做多设计空间消融，报告时间显式 semantic ID 显著提升推荐表现；摘要未给出具体数值。
- 可转化价值：可借鉴到 Agent memory 的时间感知索引，不建议作为核心 Agent 论文阅读。
- 局限：推荐系统是模板负向过滤中的相邻领域；除非关注长期用户行为模拟，否则不应高权重。
- 阅读建议：低优先级。

### 10. Co-Evolved Spiking Neural Network Ensembles via Marginal Contribution Fitness

- 链接：[arXiv](https://arxiv.org/abs/2606.13985) | [PDF](https://arxiv.org/pdf/2606.13985)
- 证据来源：PDF 正文抽取
- 问题背景：SNN 进化优化要搜索 topology 和参数，任务复杂后空间超指数增长。普通 ensemble 往往是独立训练后再组合，成员训练时不知道自己对整体的边际贡献。
- 核心方法：论文让候选 SNN 在 ensemble 中共同进化，并用每个网络对 group performance 的 marginal contribution 定义 fitness。冗余成员会被惩罚，能稳定提升整体表现的成员被奖励。
- 创新点：对多 Agent 协作训练有类比意义：不要只评估单个 agent 的绝对分数，而要评估它对团队结果的边际贡献。但论文主体是神经形态 SNN，不是 LLM agent。
- 实验与 benchmark：正文评估 classification、regression 和 reinforcement learning control 三类任务，并考虑 μCaspian neuromorphic hardware constraints。论文报告 co-evolved ensembles 在所有任务上优于 single-network evolution 和 post-hoc ensemble，control 任务提升最明显。
- 可转化价值：可作为多 Agent credit assignment 的远邻参考。
- 局限：与当前 LLM tool-use agent 距离较远，方法和评测不能直接迁移。
- 阅读建议：仅在研究 multi-agent credit assignment 时略读。

## 今日重点趋势

- 生产 Search Agent 的关键不只是 prompt，而是奖励源、上线 gate、稳定性指标和下游 ranker 消费方式的闭环一致性。
- Code Agent benchmark 正在从“题目 + 参考答案”走向“可执行验证 + 失败类型诊断 + 数据合成质量控制”。
- Agent 评测数据如果依赖模拟用户，需要把 persona、contract、trace、verification、feedback 都结构化，否则模拟数据不可审计。
- 长上下文和 memory 方向开始把实体关系、时间、结构化状态纳入检索索引，而不是只扩大 context window。

## 潜在研究 idea

- 为 Search Agent 训练一个 evidence cardinality policy：输入 query、候选 chunk 双分数、历史回答失败信号，输出“保留多少证据”和“是否继续检索”。
- 把 CoRe 的两族 promotion gate 迁移到工具使用 RL：reward-like 指标衡量任务成功，stability 指标约束调用次数、输出长度、无效重复、失败恢复。
- 设计 Code Agent benchmark synthesis pipeline：从已有可执行任务转换新语言/框架任务，用工具链验证生成样本，只保留可复现任务。
- 把 MeEvo 的 shared history 改成 Agent trajectory memory：代码、错误、测试、反思、修复理由分层保存，用于下一轮策略生成。
