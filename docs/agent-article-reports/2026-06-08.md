## General Agent

### 1. LLM-Guided Evolution for Medical Decision Pipelines
- 链接：[arXiv](https://arxiv.org/abs/2606.07342) | [PDF](https://arxiv.org/pdf/2606.07342)
- 证据来源：PDF
- 核心方法：把 urgency triage、interactive consultation、medical image classification 都重写成“对可执行 artifact 做 MAP-Elites 演化搜索”的问题，让 LLM 在推理时搜索 prompt、决策逻辑和问诊策略，而不是先做额外微调。
- 创新点：这篇论文的真正价值不在医疗，而在“把完整 decision pipeline 视作可演化程序”。这比只调 prompt 更接近 Agent runtime 的真实形态，因为优化对象已经是可执行策略而不是静态文本。
- 可转化实验：可以直接迁移到 Agent 训练里，把工具调用顺序、检索策略、重试策略、预算分配、澄清提问策略编码成 artifact，用任务成功率、成本和安全约束做 fitness。
- 局限：适配任务依赖明确的评测器或奖励函数，且目前验证域仍以医疗场景为主；如果没有稳定 evaluator，演化很容易过拟合代理指标。
- 阅读建议：本批优先级最高。它非常适合启发 General Agent 的 post-training 数据构造、程序化 policy search 和 test-time strategy discovery。

### 2. Watch, Remember, Reason: Human-View Video Understanding with MLLMs
- 链接：[arXiv](https://arxiv.org/abs/2606.07433) | [PDF](https://arxiv.org/pdf/2606.07433)
- 证据来源：PDF
- 核心方法：不是提出单一模型，而是把视频 MLLM 系统拆成 `watching / remembering / reasoning` 三种能力，再用感知表示、记忆状态、推理轨迹和最终预测四个层面组织已有方法。
- 创新点：它把长视频理解中的“稀疏证据、长程依赖、记忆保留、有限预算推理”放进一个统一框架，这种抽象比纯视频综述更贴近 agentic long-context system design。
- 可转化实验：可把这套框架映射到 browser/video agent 或多模态 computer-use agent，单独评测观察、记忆压缩、跨步推理和最终动作选择之间的误差传播。
- 局限：本质是高质量综述，不是新的 agent benchmark 或训练方法；对 Search / Code Agent 的帮助主要来自系统拆解思路，而不是直接可复用的实验脚本。
- 阅读建议：如果你近期在看长上下文记忆、轨迹压缩或多模态证据整合，这篇值得精读。

### 3. Depth over Fidelity in Fixed-Budget Noisy Evolution Strategies
- 链接：[arXiv](https://arxiv.org/abs/2606.06555) | [PDF](https://arxiv.org/pdf/2606.06555)
- 证据来源：PDF
- 核心方法：作者认为在固定评估预算下，与其花很多样本去提高单代排序精度，不如优先保留更多更新步数；具体实现是用 probabilistic elite membership、residual bootstrapping 和 probe-and-switch 处理 noisy ranking。
- 创新点：这篇论文虽然不属于 Agent 论文，但它抓住了 test-time search / policy search 的共性瓶颈，即“每次更准一点”和“多走几步”之间的预算交换。
- 可转化实验：可以把同样的深度优先思想用在 LLM self-play、prompt evolution、tool-use trajectory search，比较“更多候选步数”与“更高单步评分精度”哪个更划算。
- 局限：问题设定是 noisy black-box optimization，不是专门针对语言模型或工具调用闭环，迁移时还需要补上语义 evaluator。

## Search Agent

本批没有直接命中 Web 搜索、RAG 或浏览器轨迹级 benchmark 的强相关论文，下面三篇主要提供“可迁移的方法部件”。

### 4. SVHighlights: Towards Extremely Long Sport Video Highlight Detection
- 链接：[arXiv](https://arxiv.org/abs/2606.06926) | [PDF](https://arxiv.org/pdf/2606.06926)
- 证据来源：PDF
- 核心方法：构建超长体育视频 highlight benchmark，并提出 TF-SELECTOR，把长视频按 transcript 与镜头变化合并成 context-aware segments，再用包含 caption、transcript、audio volume 的多模态输入交给 LLM 打分。
- 迁移价值：它最有用的不是体育，而是“先做上下文保持的分段，再做高层筛选”。这和 Search Agent 处理超长网页、多段证据聚合、先切块再 rerank 的流程很像。
- 局限：任务目标仍是 highlight detection，不是事实检索或答案验证；其 segment saliency 不等于 search evidence fidelity。

### 5. MMAE: A Massive Multitask Audio Editing Benchmark
- 链接：[arXiv](https://arxiv.org/abs/2606.07229) | [PDF](https://arxiv.org/pdf/2606.07229)
- 证据来源：PDF
- 核心方法：构建一个覆盖 7 种音频模态、6 个任务复杂度层级、8 类操作的 instruction-based audio editing benchmark，并用 rubric-based evaluation 把自由形式任务拆成 17,741 个可验证标准。
- 迁移价值：对 Search Agent 最有启发的是 rubric 化评测。多轮检索、多跳核验和引用正确性其实也适合拆成结构化 criteria，而不是只看最终 answer-level 分数。
- 局限：它是音频编辑 benchmark，不是信息检索 benchmark；如果迁移到 Search Agent，需要自己重建 evidence-preservation 和 citation-faithfulness 的评分维度。

### 6. Watch, Remember, Reason: Human-View Video Understanding with MLLMs
- 关联原因：其中 `remembering` 和 `reasoning` 两部分讨论了长视频条件下如何保留关键证据、避免上下文漂移，这与 Search Agent 的长轨迹证据保持非常同构。

## Code Agent

本批也没有直接命中仓库级 patch、自动调试或 SWE-bench 类论文，因此这里重点看“底层模型/训练方法是否值得迁移到 Code Agent 体系”。

### 7. Sparsely gated tiny linear experts
- 链接：[arXiv](https://arxiv.org/abs/2606.07414) | [PDF](https://arxiv.org/pdf/2606.07414)
- 证据来源：PDF
- 核心方法：把传统 MoE 专家继续缩小到“单神经元线性专家”，并用 sparse gating 只激活极小一部分 neuron experts，目标是在相同计算量下提高语言模型 perplexity，同时增强可解释性。
- 创新点：论文最强的信号是“更强稀疏性 + 去掉专家非线性”并没有毁掉性能，反而在 isoflop 比较里提升了语言模型效果，还让前馈电路更容易解释。
- 可转化实验：如果你关心 Code Agent 底座模型，这篇值得放进“可解释 sparse FFN / MoE 变体”跟踪列表。它未必立刻改善 coding benchmark，但对后续可审计代码模型训练很有启发。
- 局限：这仍是 backbone 级论文，不是代码任务论文；距离 repo-level issue resolution 还有明显距离。

### 8. Synthetic Benchmarks Overstate Forward-Forward Scaling: Real-Data Limits of Layer-Local Training
- 链接：[arXiv](https://arxiv.org/abs/2606.06539) | [PDF](https://arxiv.org/pdf/2606.06539)
- 证据来源：PDF
- 核心方法：提出 DTG-FF 作为更强的 Forward-Forward family baseline，再系统比较 synthetic teacher-student setting 与真实图像数据上的 scaling 结果。
- 创新点：它给出的不是“新 SOTA”，而是一个很有价值的负面结论：synthetic benchmark 上 Forward-Forward 看起来会随类数增长而占优，但这种优势不能直接转移到真实数据。
- 可转化实验：对 Code Agent 很重要的启发是，任何 post-training / local-update / synthetic task 上的改进，都必须在真实仓库和真实执行反馈上复核，否则很容易高估可迁移性。
- 局限：任务域是视觉分类，不是代码；真正可迁移的是实验设计态度，而非算法本身。

### 9. A Data-Free Symbolic Regression Approach for Solving Equations
- 链接：[arXiv](https://arxiv.org/abs/2606.07152) | [PDF](https://arxiv.org/pdf/2606.07152)
- 证据来源：PDF
- 关联原因：论文把“从方程直接恢复显式符号表达式”建模成 differentiable symbolic model 优化。它不是 Code Agent 论文，但对 program synthesis、formal reasoning 和显式程序结构搜索有一定启发。
- 局限：与实际代码仓库任务距离较远，优先级明显低于前两篇。

## 今日重点趋势

1. 这批最值得提炼的主线是“把复杂决策流水线改写成可搜索、可演化、可分段评测的对象”，而不是继续堆更大的 end-to-end 模型。
2. 长上下文问题正在从“塞进更多 token”转向“显式管理记忆状态和证据分段”。`Watch, Remember, Reason` 与 `SVHighlights` 都体现了这一点。
3. 评测设计继续朝结构化标准前进。`MMAE` 的 rubric-based evaluation 和 `LLM-Guided Evolution` 的 task-specific fitness，都说明 agent 研究越来越依赖细粒度 evaluator，而不是单一最终分数。

## 潜在研究 Idea

1. 把 `LLM-Guided Evolution` 的 artifact search 迁移到 Agent runtime：搜索工具调用顺序、重试条件、澄清问题模板和预算路由。
2. 把 `Depth over Fidelity` 的预算观迁移到 Agent self-improvement：固定预算下优先探索更多策略步数，而不是为单个候选做过度精确评审。
3. 借鉴 `MMAE` 的 rubric 分解，为 Search / Code Agent 构建“多轮检索正确性”“引用保真”“patch 可执行性”的细粒度自动评分表。
4. 把 `Watch, Remember, Reason` 的三段式拆解映射到 browser / code agent，显式区分观察压缩、工作记忆和推理控制。
