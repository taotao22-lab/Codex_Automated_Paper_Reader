### General Agent

`Remember When It Matters`、`Feedback Manipulation Regularization` 和 `Playing ZendoWorld` 最值得优先看。前者把 memory agent 设计成独立的主动干预模块，强调选择性提醒而不是全量记忆暴露；FMR 把 evaluative feedback 用作离线 imitation learning 的对齐正则，适合迁移到顺序决策式 agent alignment；ZendoWorld 则把感知、归纳和主动实验放进同一个交互环境，暴露当前 VLM agent “会分类但不会设计有效实验”的问题。

### Search Agent

`PolyUQuest` 是今天最直接的 Search/RAG 论文。它把网页链接、DOM 层级和实体关系放进异构图，再用路由器选择块检索、跨页导航或实体多跳推理，并保留可追溯证据路径。`Ideas Have Genomes` 虽然不是传统搜索系统，但它把 scientific lineage reasoning 做成可评测 benchmark，对文献检索、related work agent 和研究 idea agent 很有迁移价值。

### Code Agent

`Compete Then Collaborate` 和 `CausalDS` 是今天最重要的 Code/Data Agent 信号。前者显示同一批执行验证过的编程题，用 SFT 模仿会退化，而用 RLVR/GRPO 当可验证 reward 环境能提升；后者把结构因果模型、数据文件、自然语言场景、工具使用、不可识别时 abstain 合成到同一 benchmark 中，适合作为数据科学 agent 的闭环评测。

### 1. Remember When It Matters: Proactive Memory Agent for Long-Horizon Agents

- 类别：General Agent / Long-Horizon Memory
- 链接：[2607.08716](https://arxiv.org/abs/2607.08716)
- 综合评分：4.95

**问题背景与核心方法**：论文解决的是长程任务里的 `behavioral state decay`：任务要求、环境事实、失败尝试、诊断结论和未完成子目标散落在越来越长的轨迹里，action agent 在关键时刻看过却不用。作者没有改 action agent，而是在旁边放一个 memory agent。它持续观察轨迹，维护结构化 memory bank，并在判断“这条状态会影响下一步决策”时注入一条短提醒；如果不需要，就保持沉默。记忆被分成 knowledge memory 和 procedural memory，前者保存任务约束、路径、环境事实，后者保存失败命令、有效修复、被排除的假设和调试经验。

**创新点**：关键不是“把更多历史塞回上下文”，而是把 memory 作为主动干预策略。论文的 ablation 说明 full-bank context、always inject、无 bank 的 advisor-like guidance 和通用 retrieval 都不如“维护执行状态 + 选择性提醒/沉默”。这对真实 tool-use agent 很可迁移：记忆模块不必替代主 agent，只要在决策前重激活被忽略的状态。

**实验与 benchmark**：PDF 正文报告 Terminal-Bench 2.0 和 τ2-Bench。Terminal-Bench 2.0 上，Sonnet 4.5 action agent 从 37.6% pass@1 提升到 45.9%，+8.3 pp；Opus 4.6 从 43.5% 到 45.9%，+2.4 pp。τ2-Bench 上，Sonnet 4.5 task-weighted average 从 55.0% 到 61.8%，+6.8 pp；Opus 4.6 从 66.2% 到 68.7%，+2.5 pp。作者还用 SETA 对 Qwen3.5-27B memory policy 做 SFT 和 GRPO，观察到 validation reward 提升和到 Terminal-Bench 的部分迁移。

**可迁移价值与限制**：适合直接迁移为 agent harness 的外置 memory controller：不改主模型，只训练何时写 memory、何时删 memory、何时注入提醒。限制是 memory agent 主要依赖强闭源模型做决策，开源 memory policy 仍是 early step；不同 benchmark 对 memory 的收益差异较大，需要按任务域调策略。

### 2. CausalDS: Benchmarking Causal Reasoning in Data-Science Agents

- 类别：Code/Data Agent / Tool-use Benchmark
- 链接：[2607.08093](https://arxiv.org/abs/2607.08093)
- 综合评分：4.75

**问题背景与核心方法**：现有数据科学 agent benchmark 往往测试数据处理和编码，但没有清晰的因果 ground truth；因果推理 benchmark 又常常停留在符号题，没有真实数据分析和工具调用。CausalDS 生成一个“场景”：采样结构因果模型，生成观测数据，再配一段合成但像真实领域的自然语言故事。每个任务覆盖 Pearl 三层，从预测到干预再到反事实，并且故意加入 proxy、hard proxy、噪声观测和不可识别问题。

**创新点**：它把 causal reasoning、data science coding、uncertainty quantification、tool use 和 abstention 放到同一可评分环境。特别重要的是 abstain 被当作一等结果：如果问题在因果上不可识别，模型应该拒答而不是编一个数字。这对训练 agent 的“知道什么时候不能回答”很有价值。

**实验与 benchmark**：PDF 报告 realistic-composition exam：100 个 scene，包含 28 个 Rung 1、51 个 Rung 2、21 个 Rung 3 任务。评测模型包括 Claude Opus 4.8、Gemini 3.1 Pro、GPT-5.5、Qwen 3.6 35B、Kimi K2.6、Gemma 4 26B。表格显示 Claude Opus 4.8 和 GPT-5.5 总 pass rate 均为 82.4%，Gemini 3.1 Pro 为 76.5%，Kimi 65.7%，Qwen 63.2%，Gemma 55.9%。论文指出区分度集中在 Rung 2 和不确定性量化，95% ATE interval coverage 出现明显过度自信；工具调用策略也分化很大，GPT-5.5 和 Claude 接近 one-shot，Kimi/Qwen/Gemini 更依赖多轮 bash。

**可迁移价值与限制**：可作为 Data Agent 后训练任务生成器：让模型读 story、读文件、写代码、估计因果量、判断不可识别并校准置信区间。限制是场景仍为合成生成，真实数据分布和业务噪声需要额外验证。

### 3. Compete Then Collaborate: Frontier AI Teachers Build a Verifiable Curriculum to Improve a Coding Student Beyond Imitation

- 类别：Code Agent / Verifiable Curriculum / RLVR
- 链接：[2607.08255](https://arxiv.org/abs/2607.08255)
- 综合评分：4.7

**问题背景与核心方法**：多教师蒸馏常把强模型输出合并给小模型模仿，但编程任务里“答案看起来好”不等于对。论文先让 Claude、Codex-GPT、Grok、Gemini 四个 teacher 在 MBPP 和 code_contests 上竞争，judge 不是 LLM 偏好，而是隐藏单测和 stdin/stdout 执行检查。通过执行验证留下可用解，然后把同一批 verified curriculum 用两种方式给 Qwen2.5-Coder 学生：SFT 模仿，或作为 GRPO/RLVR 的 verifiable reward environment。

**创新点**：最重要的结论是“同一批数据，模仿和强化学习方向相反”。论文把 teacher collaboration 的价值从“合并答案”改成“共同构造可验证环境”。这和 Code Agent 后训练高度相关：不要把 teacher trace 只当 SFT 语料，更应把它转成可执行题、隐藏测试、reward 和失败反馈。

**实验与 benchmark**：PDF 报告 MBPP 200 题 teacher 接近饱和，hard code_contests 150 题更有区分度。学生侧结果更关键：在 held-out competition problems 上，SFT union 从 5.9% 降到 2.9%；RLVR/GRPO 200 steps 从 5.9% 到 7.4%；v2 1000 steps peak 到 8.8%，相对提升 +49%，后期略回落到 7.4%。论文也说明 MBPP 可能有 train-leak 和饱和风险，因此教师排名主要依赖更难的 code_contests。

**可迁移价值与限制**：适合把 coding teacher pipeline 设计成“执行验证 + reward 环境”，而不是蒸馏最终答案。限制是实验规模偏小，且 MBPP 公开数据污染风险较高；不过其方向对 agentic code training 很清楚。

### 4. PolyUQuest: Verifiable Structure-Aware Web RAG over Heterogeneous Graphs

- 类别：Search Agent / Web RAG / Provenance
- 链接：[2607.08269](https://arxiv.org/abs/2607.08269)
- 综合评分：4.65

**问题背景与核心方法**：普通 RAG 把网页切成平面文本，丢掉 HTML 的层级、标题路径、链接拓扑和跨页实体关系。PolyUQuest 把官方网页构造成异构图：页面之间有 hyperlink topology，页面内部有 DOM hierarchy，跨页面还有 entity-relation knowledge。查询先经过 two-tier router，选择 direct block retrieval、cross-page graph traversal 或 multi-hop entity reasoning。每个回答都带 source page、heading path、entity links 和 retrieval trace，方便检查证据。

**创新点**：它把 Search Agent 的“浏览网页结构”变成图检索和证据路径，而不是让 LLM 暴力读大上下文。对 web QA 来说，这比 chunk embedding 更接近真实网页导航：某些问题要找页面块，某些问题要沿链接跨页找，某些问题要围绕实体关系多跳。

**实验与 benchmark**：PDF 显示 demo 使用 PolyU 4,240 个英文网页、31,086 个 DOM blocks、29,119 个 entities、37,680 条 relations，并在 PolyU-Web 300 questions 上评估 correctness、coverage、faithfulness、query tokens。PolyUQuest 得到 Corr. 0.644、Cov. 0.649、Faith. 0.921，高于 ChunkRAG、HtmlRAG、FastGraphRAG、LightRAG 等；同时平均每 query 2,968 tokens，比 LightRAG 少约 10 倍，比 HtmlRAG 少 26%。Ablation 显示去掉 DOM blocks 会让 correctness 降 8.1 点、coverage 降 13.9 点。

**可迁移价值与限制**：可迁移为 Search Agent 的结构化网页索引器和可验证 citation layer。限制是 benchmark 来源于单个大学站点，结构模式、更新频率和跨域网页质量还需要更广泛验证。

### 5. From Prompts to Contracts: Harness Engineering for Auditable Enterprise LLM Agents

- 类别：General Agent / Production Harness / Auditability
- 链接：[2607.08028](https://arxiv.org/abs/2607.08028)
- 综合评分：4.6

**问题背景与核心方法**：企业 LLM agent 原型常把行为塞进 prompt 和 retrieval context；一旦产品化，就需要 source boundary、entity routing、answer contract、trace、validator 和 fallback。论文提出 harness-engineering：把确定性行为移出 prompt，放入代码、manifest、schema、source-backed claims、validation artifacts 和 deterministic fallback；LLM 只在 replaceable composition boundary 里负责读结构化证据并组织面向读者的语言。

**创新点**：它强调“prompt 不是 guardrail”。真正可审计的 agent 需要 versioned contracts 和 code-owned validation gate。这个设计对 production agent 很实际：把 source selection、claim coverage、trace envelope、leakage prevention、recommendation-language checks 都做成可测试契约，而不是靠模型记住规则。

**实验与 benchmark**：论文在五个韩国企业集团、25 家上市公司 public-data slice 上做固定场景验证。模型替换实验在 Claude Sonnet 4、GPT-4.1 mini、Gemini 2.5 Flash 上跑 30 个 fixed scenarios、每个 3 次，共 270 runs；enforcement-layer ablation 在 40 scenarios、每条件 120 runs 上比较 harness gate、prompt-only 和 external guardrail。PDF 结论称 harness 保持 source、claim、routing、trace、leakage、follow-up、output contract，prompt-only 会让违规输出到达读者，bolt-on guardrail 会过度拒答。

**可迁移价值与限制**：适合当作 agent 产品化的工程模板：把 prompt 原型重构成 source/claim/contract/validator/fallback。限制是评测集中在 bounded public-data slice，最终 claim correctness 仍需要专家审查。

### 6. Feedback Manipulation Regularization: Enabling Offline Agent Alignment for Imitation Learning

- 类别：General Agent / Offline Alignment / Sequential Decision Making
- 链接：[2607.07859](https://arxiv.org/abs/2607.07859)
- 综合评分：4.55

**问题背景与核心方法**：许多 alignment 方法围绕语言生成的 contextual bandit 展开，但真实 agent 是顺序决策：动作影响后续状态，演示和反馈也应被联合利用。FMR 把 evaluative feedback 当作 corrective signal，用正则项调整 imitation learning policy，使其少走偏人类价值的轨迹。它是 algorithm-agnostic，可以叠加到 BC、DemoDICE、IQL、ReCOIL 等 imitation/offline RL 方法上。

**创新点**：它不是再训练一个 reward model 后做在线 RL，而是在离线 imitation setting 里把反馈转成“少模仿错误倾向”的约束。对 agent post-training 的启发是：人类或 verifier 的评语不只用来排序回答，也可以在 sequential policy learning 中改变 action distribution。

**实验与 benchmark**：PDF 报告使用 Safety Gymnasium、Gymnasium 和 MuJoCo，包含 3D navigation 和 Hopper/Swimmer/Walker2D velocity-restricted locomotion。cost 被用作 misalignment 评价信号。导航任务中，ReCOIL+FMR 在 PathM 上 misalignment 最多降低 92%，PathBB 上降低 67%；velocity 任务中 FMR variants 在不同 data ratio 下更稳定，SlowSwim 上最高报告 98% misalignment reduction。

**可迁移价值与限制**：适合迁移到 tool-use agent 的离线轨迹修正：把失败执行、违规工具调用、错误恢复作为 evaluative feedback，训练模型降低这类行为概率。限制是当前实验在控制/导航环境，不是 LLM tool-use 环境；语言 agent 需要重新定义 cost 和反馈格式。

### 7. Playing ZendoWorld: Challenging AI Agents on Active Visual Concept Induction

- 类别：General Agent / Active Experimentation / Visual Reasoning
- 链接：[2607.08233](https://arxiv.org/abs/2607.08233)
- 综合评分：4.45

**问题背景与核心方法**：很多视觉推理 benchmark 只测对已有样本打标签，但科学发现式 agent 还要提出实验来区分假设。ZendoWorld 借鉴归纳逻辑游戏 Zendo：agent 观察带标签的视觉场景，推断隐藏规则，再生成新场景测试假设。环境同时包含 raw visual input、formal DSL 和 precise feedback，使 perception、induction、experimentation 在同一 loop 中可测。

**创新点**：论文直接揭示“label accuracy 不等于 rule recovery”。VLM agent 可以正确判断很多已有例子，却提出近乎无信息量的新实验，不能主动降低 hypothesis uncertainty。这个发现对 Search/Research Agent 很重要：评价 agent 不能只看最终回答，还要看它提出的查询或实验是否真的能区分假设。

**实验与 benchmark**：PDF 报告 22 games、5 seeds、4 agents，共 440 runs；完整评估使用 321 GPU-hours 和约 155M GPT-5-mini API tokens。指标包括 win rate、turn count、hypothesis structural F1 和 expected information gain。附录还用 Qwen3.5-27B、Gemma-4-31B-it、Claude Opus 4.8、Gemini 3.1 Pro preview 替换 VLM backbone 做 ablation。

**可迁移价值与限制**：适合转化为 agentic evaluation：让模型不仅回答，还要主动设计下一步观测。限制是视觉 DSL 游戏与实际网页/代码环境有距离，且实验成本较高。

### 8. Infinity-Parser2 Technical Report

- 类别：General/Search Agent Infrastructure / Document Parsing / Multi-task RL
- 链接：[2607.07836](https://arxiv.org/abs/2607.07836)
- 综合评分：4.45

**问题背景与核心方法**：复杂文档解析不仅是 OCR，还包括版面、表格、公式、图表、化学式、阅读顺序和 document VQA。Infinity-Parser2 用 controllable rendering framework 和 iterative refinement loop 合成 Infinity-Doc2-5M 双语数据，并用 multi-task reinforcement learning 统一优化 document parsing、layout analysis、table parsing、math formula parsing、chart parsing、chemical formula parsing、document VQA 和 general multimodal understanding。

**创新点**：对 agent 的价值在于文档理解基础设施。Search/Research Agent 若要稳定读 PDF、报告、表格和图表，需要可验证的结构化解析，而不是只抽纯文本。论文的 reward system 试图把空间对齐、结构忠实度和跨任务迁移纳入同一个优化信号。

**实验与 benchmark**：PDF 报告 Infinity-Parser2-Pro 在 olmOCR-Bench 达 87.6%，ParseBench 达 74.3%，超过 DeepSeek-OCR-2、PaddleOCR-VL-1.5、MinerU2.5；Flash 版本相对 Infinity-Parser-7B throughput 从 441 tokens/s 到 1,624 tokens/s，约 3.68 倍。论文还列出 PubTabNet、FinTabNet、MMTab-HTML、im2latex、UniMER、HME、CROHME、ChartSFT、ChartQA、DocLayNet、OmniDocBench 等多类评测/训练来源。

**可迁移价值与限制**：可作为文档型 Search Agent 的 parser layer 和 reward design 参考。限制是技术报告很长，部分表格由 PDF 抽取不稳定；此外模型专门化带来一些纯数学/逻辑 benchmark 的下降，说明 parser 强化会有能力取舍。

### 9. Ideas Have Genomes: Benchmarking Scientific Lineage Reasoning and Lineage-Grounded Idea Generation

- 类别：Search/Research Agent / Scientific Lineage Benchmark
- 链接：[2607.08758](https://arxiv.org/abs/2607.08758)
- 综合评分：4.4

**问题背景与核心方法**：研究 idea 不是凭空生成，而是继承、修复、重组前人机制。论文提出 IdeaGene-Bench，用 Idea Genome 表示论文或提案中的最小 typed、evidence-grounded idea object，再用 GenomeDiff 对齐 inherited、mutated、lost、external import、novel insertion 等变化。benchmark 包括 IG-Exam 和 IG-Arena：前者测 lineage reasoning，后者测 lineage-grounded idea generation。

**创新点**：它把 research agent 的 related work 能力从“检索相似论文”提升为“解释机制继承关系”。对自动论文阅读和 idea generation 来说，这比 citation graph 或 embedding 相似度更接近人类写 related work 的核心能力。

**实验与 benchmark**：PDF 摘要和正文显示 benchmark 包含 1,961 条 golden lineage traces、1,085 个 curated Idea Genome objects、920 个 pairwise GenomeDiff records，覆盖 10 个科学领域。IG-Exam 有 42 task types、1,029 instances；IG-Arena 用 Population-Evolution Score (PES) 衡量生成提案是否能作为给定 lineage population 的合理后代。实验覆盖 14 个 LLM-based scientists，最强系统 lineage reasoning exact accuracy 只有 27.3%，暴露组合式瓶颈；PES 的 inter-judge agreement Krippendorff's alpha 为 0.74，人类一致性约 80%。

**可迁移价值与限制**：适合做 Research/Search Agent 的高级评测：不仅问“找到了哪些论文”，还问“机制从哪里继承、哪里变异、能否产生合理下一步”。限制是 Idea Genome 标注成本高，且评分依赖结构化定义和 judge panel。

### 10. Game Theory Driven Multi-Agent Framework Mitigates Language Model Hallucination

- 类别：General Agent / Multi-agent Data Synthesis / Domain Reasoning
- 链接：[2607.08403](https://arxiv.org/abs/2607.08403)
- 综合评分：4.1

**问题背景与核心方法**：轻量 LLM 在规则密集的科学领域容易模仿语言模式而非内化公理约束，导致 hallucination。G-Frame 用 team game 和 Bayesian game 组织多个 LLM agent：一方面做任务分解、数据清洗、数据增强、CoT/QA 生成；另一方面把 checkpoint 评估作为 payoff，用 Bayesian update 调整训练策略、学习率和数据增强。

**创新点**：论文的可迁移点不是化学本身，而是“多 agent 生成训练数据 + 用评价 payoff 自适应训练”的流水线。它把 teacher/student/supervisor 式 QA 生成、CoT 蒸馏、异步并发、checkpoint judge 和训练策略更新合在一起，形成面向 domain model 的自动数据-训练闭环。

**实验与 benchmark**：PDF 报告生成 363,045 条 chemical CoT 和 199,589 条 QA pairs，训练 7B OmniChem。ChemJudge 是 471-question test，由 Gemini 3.1 Pro 评判；论文称 hallucination rate 相对 base model 降低 79.46%。在 ThChem 1.0、ThChem 2.0、ChemBench 上分别报告 79.45%、62.08%、49.82% accuracy。系统吞吐方面，异步模式相对同步提升约 22 倍，并高于固定并发 50/100 的设置。

**可迁移价值与限制**：可借鉴到 agent 训练数据合成：用多个角色生成、审查、修订，并让评价信号反过来调训练流程。限制是领域强绑定化学，许多结果依赖自建 ChemJudge 和 Gemini judge；泛化到通用 agent 需要换成真实工具执行或可验证 reward。

## 今日重点趋势

1. 长程 agent 的记忆正在从被动检索转向主动干预。最有价值的不是“记住所有东西”，而是在关键决策点重新激活被忽略的执行状态。
2. Code Agent 后训练不应只做 teacher answer SFT。执行验证、隐藏测试、RLVR/GRPO 和 reward environment 是更强信号。
3. Search/RAG 的结构化证据路径正在变重要。DOM、链接、实体关系、heading path 和 citation trace 比平面 chunk 更适合可审计回答。
4. Agent benchmark 正在引入 abstention、uncertainty calibration、实验设计质量和 lineage reasoning，评价目标从最终答案扩展到行动策略。

## 潜在研究 idea

- 设计一个 proactive memory policy 数据集：从 Terminal-Bench、SWE-bench agent trace 和真实工具日志中标注“此刻应该提醒哪条状态/保持沉默”。
- 把 coding teacher 生成结果转成 RLVR 环境：每条样本包含题面、隐藏测试、失败输出、修复奖励，而不是只保留最终代码。
- 为 Web/Search Agent 构建 DOM-aware retrieval reward：奖励引用 heading path、跨页实体路径和 claim-level provenance，而不是只奖励语义相似 chunk。
- 用 IdeaGene 风格的结构化 lineage 评测自动论文阅读器：要求模型给出“继承了哪篇论文的哪项机制、改了什么、为什么是合理下一步”。
