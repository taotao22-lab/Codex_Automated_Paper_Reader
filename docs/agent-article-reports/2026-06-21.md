## General Agent

### 1. Do LLM-Based SRE Agents Know When They Can't Fix It? Measuring Operational Calibration Under Degraded Telemetry

- 链接：[OpenReview](https://openreview.net/forum?id=MwbxDJcROt) / [PDF](https://openreview.net/pdf/372230bd3eb58bce49b18c6cf0f20828fac2389e.pdf)
- 问题背景：现有 AIOps/SRE agent benchmark 多数只问“能不能修好事故”，但真实自动化运维里更危险的问题是观测信号损坏时 agent 是否知道不该继续执行。论文把这个问题定义为 operational calibration：行动质量应随日志、指标、trace 的可靠性下降而下降，必要时升级给人。
- 核心方法：作者提出 OCULUS，在 Cloud-OpsBench 上系统性破坏 telemetry。破坏维度包括日志、指标和 trace 的缺失或污染，严重程度从 clean 到 50-80% 信号损失。评测不只看 resolve，还看 agent 是否继续行动、是否升级、是否在输入质量下降时改变策略。
- 创新点：它把“校准”从分类置信度扩展到可执行 agent 的行动层。这个视角对 Code Agent/SRE Agent 很直接：执行工具前应有输入质量估计、行动风险估计和 abstain/escalate 策略，而不是只靠 prompt 要求“谨慎”。
- 实验与 benchmark：论文报告在 Cloud-OpsBench 的 754 个 fault cases 上进行 16,800 次运行，覆盖 7 种模型配置，包括 frontier API、开源模型和 multi-agent 系统。关键结果包括 action-taking tendency 与 safety 强负相关 `r=-0.997, p<0.001`，最佳 Operational F1 为 0.70；只有少数配置在统计校正后表现出显著 positive operational calibration；模型更容易识别 missing data，却不善于识别 corrupted data。
- 可迁移价值：可直接转成 Agent 训练数据构造思路：给同一个任务生成 clean/missing/corrupted 多版本观测，监督模型选择 act/ask/escalate/inspect。对代码执行闭环也可加入“测试结果冲突、日志截断、环境不一致”情形。
- 局限：目前是在 SRE 场景和 Cloud-OpsBench 上验证，是否泛化到浏览器、代码仓库修复、搜索任务需要新环境。
- 阅读建议：强烈建议精读，尤其是 degradation taxonomy、Operational F1 和 prompt-dependent safety 相关实验。
- 证据来源：PDF。

### 2. Data Selection for Reward Labeling in Limited-Feedback Reinforcement Learning

- 链接：[OpenReview](https://openreview.net/forum?id=N8OlHGeHSy) / [PDF](https://openreview.net/pdf/972b7f689a15b9780d699cf661762b8228ee65d6.pdf)
- 问题背景：Agent post-training/RLHF 常见瓶颈不是轨迹不够，而是高质量 reward label 太贵。论文把问题抽象为 reward selection：在只能标注少量样本奖励时，应该标哪些状态/轨迹才能最大化策略改进。
- 核心方法：它比较三类选择策略：仅依赖 reward-free 信息的启发式方法、近似最优但代价高的选择方法、以及在成本和性能之间折中的方法。核心发现是有价值的奖励点不只是最优轨迹上的点，还包括 agent 偏离后能恢复到近优行为的关键点。
- 创新点：相比“多标困难样本”或“均匀采样偏好”，该工作把有限反馈建成 RL 问题内部的选择任务，强调 recovery reward 对长期闭环行为的价值。这对工具使用 agent 很重要：不只标成功路径，也要标错误调用后如何恢复。
- 实验与 benchmark：PDF 中实验覆盖 6 个原型域以及 MinAtar 的 Breakout、Freeway、Seaquest、Asterix；还包含 FrozenLake、CliffWalk 等 Gymnasium benchmark。指标包括 average episode return、optimality gap 和按状态数比例定义的 feedback budget。
- 可迁移价值：可用于设计 Agent 训练数据预算：优先标注“决策分叉点”“错误后恢复点”“工具调用失败后的下一步”，而不是只对最终答案打分。
- 局限：论文主要在可控 RL 环境里验证，LLM agent 的语言轨迹和工具执行成本需要重新建模。
- 阅读建议：适合和 DeepSeek/agentic RL 数据配方一起看，重点看 reward acquisition budget 和 heuristic selection。
- 证据来源：PDF。

### 3. Unlocking Out-of-Distribution Generalization in Transformers via Latent Space Reasoning

- 链接：[OpenReview](https://openreview.net/forum?id=8bFgEyRLrO) / [PDF](https://openreview.net/pdf/8fee568dd5facb8d7d21a9cbbe6e778491a2dc7b.pdf)
- 问题背景：Transformer 在训练分布内可以学会推理格式，但对更长、更复杂组合结构的 OOD 泛化仍不稳定。论文用 GSM8K-style modular arithmetic circuit-evaluation 作为可控 testbed，研究 length/generalization 机制。
- 核心方法：作者组合四个机制：input-adaptive recurrence、algorithmic supervision、discrete bottleneck anchored latent representations、explicit error-correction。直观说，它让模型不是一次性在隐空间里“猜答案”，而是允许按输入复杂度增加内部迭代，并用离散锚点和纠错机制约束中间状态。
- 创新点：它不只是加 CoT，而是把推理过程搬到 latent space 的结构约束里。对 General Agent 来说，可借鉴为“工具调用前的隐式状态机”：规划、执行、检查、纠错各自有可监督的中间表示。
- 实验与 benchmark：benchmark 是可调规模/深度的 modular arithmetic circuit evaluation，作者说明其与 GSM8K 数学推理 benchmark 的组合泛化问题相似。PDF 提到标准 CoT 在训练内有效，但测试复杂度超出训练区间后性能快速退化；四个机制共同改善 OOD 泛化。
- 可迁移价值：适合启发长程任务的训练协议，尤其是对分解、递归执行和错误修正的结构化监督。
- 局限：任务是合成数学电路，不等价于真实浏览器/代码/搜索环境；工程上需要证明机制能扩展到自然语言 agent。
- 阅读建议：中高优先级，重点看四个机制的 ablation 和 mechanistic interpretability 分析。
- 证据来源：PDF。

### 4. ESLM: Risk-Averse Selective Language Modeling with Hierarchical Batch Selection

- 链接：[OpenReview](https://openreview.net/forum?id=vcynsg6rMN) / [PDF](https://openreview.net/pdf/7703c919dda01d1807f79dfdd90919df2adb099c.pdf)
- 问题背景：预训练/继续训练中大量 token 对学习贡献很小，统一计算 loss 会浪费 FLOPs。对 Agent post-training 来说，类似问题是大量简单轨迹会稀释关键失败/恢复样本。
- 核心方法：ESLM 两阶段选择训练信号：先用 shallow early-exit pass 在 instance 级计算 loss/entropy 等 proxy，用 value-at-risk 阈值保留高价值样本；再在 token 级用 risk-aware thresholding 进行 loss shaping。论文把它解释成模型和 masking adversary 的 bilevel game，连接到 conditional value-at-risk 最小化。
- 创新点：它把“选择哪些数据训练”在线化、层次化，并把风险最坏部分作为优化目标。迁移到 Agent 数据时，可以把高风险 token 替换成高风险 step、tool call 或 observation span。
- 实验与 benchmark：PDF 中实验使用 GPT-2 pretraining，跟踪到目标 validation perplexity 所需 FLOPs、validation loss/FLOPs 或 iterations，以及 lm-eval-harness 的 QA/reasoning/generation 下游指标；片段中可见 HellaSwag 5-shot accuracy 曲线。
- 可迁移价值：可作为 agentic post-training 的样本调度机制：优先训练失败率高、信息熵高、恢复价值高的轨迹片段。
- 局限：主要是语言建模预训练，不是工具调用策略训练；proxy 是否能代表 agent 行动价值需要验证。
- 阅读建议：适合关注数据效率和训练调度的人精读。
- 证据来源：PDF。

### 5. Which Examples Preserve Decisions? Verified Inverse Design for Model Selection

- 链接：[OpenReview](https://openreview.net/forum?id=rWz4DKGsw9) / [PDF](https://openreview.net/pdf/8b84b2c1b1f5214e895d76a809fbdee6114a331e.pdf)
- 问题背景：模型选择常依赖多个场景、切片和鲁棒性测试，但很少问“哪些训练/评测证据足以保持同一个 winner map”。这对 agent benchmark 很相关：我们需要知道哪些任务真正决定了模型排序。
- 核心方法：论文提出 DPA 和 VERDICT。它先用局部 decision-response operator 与 trust-region surrogate 提出低成本数据设计，再通过重新运行固定 protocol 验证候选是否在指定 margin 下保留原有 winner。
- 创新点：它把数据子集选择目标从“近似整体性能”改成“保持决策结果”。对 agent 评测集压缩和回归测试选择很有价值，尤其是维护每日/每周 benchmark 时筛选最能守住模型排序的样本。
- 实验与 benchmark：PDF 说明在 verification feasible 的 setting 中评估，包括 train-and-verify 的 decision-preserving subset selection 和 downstream subset selection for evaluation。摘要未给具体公开 benchmark 名称，正文片段强调固定候选模型、训练流程、validation scenarios、decision convention。
- 可迁移价值：可用于构造小而硬的 Agent regression suite：只保留能稳定区分模型/训练配方的任务。
- 局限：需要能反复 retrain/verify；对昂贵 LLM agent 可能只能做近似。
- 阅读建议：作为评测集维护方法读，不必作为每日最高优先级训练论文。
- 证据来源：PDF。

### 6. Dynamically Scaled Activation Steering

- 链接：[OpenReview](https://openreview.net/forum?id=EXKv1KMNtR) / [PDF](https://openreview.net/pdf/8b422442521431102697ec2d0145e38c9bbc5e15.pdf)
- 问题背景：Activation steering 能控制模型行为，但固定强度干预会在不需要 steering 的输入上损害模型能力。Agent 中也有类似问题：安全/风格/工具偏置如果始终强行施加，会伤害正常任务。
- 核心方法：DSAS 将“何时 steering”和“如何 steering”解耦。它根据层和输入动态计算 steering strength，只在检测到目标行为域时加强干预；也可以与 steering function 一起端到端优化。
- 创新点：它提供了条件化的内部控制接口：不是全局改变模型，而是根据上下文给不同 token/层不同干预强度。对 Agent 可以迁移成“仅在高风险工具、可疑网页、冲突证据时触发强约束”。
- 实验与 benchmark：PDF 中语言模型实验使用 RealToxicityPrompts 训练，TET 测试，RoBERTa toxicity classifier 评估毒性；同时用 20,000 Wikipedia sentences 的 perplexity 和 MMLU 5-shot accuracy 检查能力保持。模型包括 Qwen 2.5 1.5B/7B 和 Gemma 2 2B，steering 方法包括 CAA、ITI、LinEAS；还演示了 text-to-image diffusion 的概念调制。
- 可迁移价值：适合启发 agent safety controller 或 runtime guard 的“按风险动态加权”设计。
- 局限：还不是直接的 agent/tool-use 方法；主要验证在毒性缓解和 T2I 概念控制。
- 阅读建议：中等优先级，适合与 inference-time control 方向关联阅读。
- 证据来源：PDF。

### 7. Objective-Behavior Alignment: Diagnostics for MORL Policy Selection

- 链接：[OpenReview](https://openreview.net/forum?id=hfnMLNCCYz) / [PDF](https://openreview.net/pdf/bf1b457fe3fff5146728d0124db43bb88ff69add.pdf)
- 问题背景：多目标 RL 常展示 Pareto front 的 value vectors，但相近的回报向量可能对应完全不同的行为轨迹。Agent 对齐也有类似风险：指标看起来接近，实际行动风格差异很大。
- 核心方法：作者提出 objective-behavior alignment 诊断工作流，自动突出 Pareto front 上目标值无法揭示的行为差异，提供定量和可视化工具辅助选择 policy。
- 创新点：把选择 policy 的依据从最终 reward 扩展到 trajectory behavior。对 Agent 来说，可用于比较“同样成功率”的模型是否采取了更安全、更可解释、更少副作用的工具路径。
- 实验与 benchmark：PDF 摘要和正文片段提到在 simple grid examples 验证，并扩展到 continuous control benchmarks；摘要未列具体环境名。
- 可迁移价值：适合转化为 Agent trajectory diagnostics，例如对同分模型比较工具调用次数、回滚次数、无效搜索比例和风险动作。
- 局限：非 LLM agent 论文，迁移需要定义行为距离。
- 阅读建议：作为评测诊断补充阅读。
- 证据来源：PDF。

### 8. Generative Retrieval via Diffusion Transformer with Metric-Ordered Sequence Training and Hybrid-Policy Preference Optimization

- 链接：[OpenReview](https://openreview.net/forum?id=aUX1pxM5lF) / [PDF](https://openreview.net/pdf/716ddfe6c812a8fa10f3b0b15a583c6fc6f4922a.pdf)
- 问题背景：传统 embedding retrieval 往往只最大化相似度，但生产环境常需要“保持 seed pattern，同时提升目标属性”。例如 Search Agent 不只是找相似网页，还要沿着某条证据链找到更有用、更可信、更满足约束的结果。
- 核心方法：MO-DiT+HPPO 把检索建成连续生成问题：模型读取 item embedding 序列，生成一个或多个 query embedding 用于近邻搜索。训练分为 raw-sequence pretraining、metric-ordered continuation pretraining、domain-specific SFT 和 Hybrid-Policy Preference Optimization。HPPO 每轮混合 deterministic high-metric candidates 与当前 policy 生成的 candidates，用真实线上 Joint@K 打标签，再做 reference-anchored preference optimization。
- 创新点：最重要的是 metric-ordered trajectory：稀疏线上标签被改造成“从低目标密度到高目标密度”的改进轨迹，让模型学习检索改进方向，而不只是静态排序。Pareto pair filter 只接受不降低 same-pattern purity 的 preference pair，避免只为指标牺牲主题一致性。
- 实验与 benchmark：PDF 报告四个大规模 attribute domains，采用严格 item-holdout 与 pattern-holdout。Metric-ordered DiT 相比强 pretrained generative retriever 提升 primary intersection metric，HPPO 进一步提升；7/8 个 domain-split cells 上 paired-bootstrap 显著，最难的 pattern split 边际持平。核心指标包括 Joint@K 和 same-pattern share。
- 可迁移价值：可直接迁移到 Search Agent 的查询改写/检索轨迹训练：把搜索过程标成“保持问题意图 + 提升证据质量”的 preference pairs，而不是只训练最终答案。
- 局限：论文面向 embedding item retrieval，不是网页浏览和引用验证；需要把 Joint@K 类指标替换为证据覆盖、引用可信度和答案可验证性。
- 阅读建议：Search Agent 方向今日最值得精读。
- 证据来源：PDF。

## 今日重点趋势

1. Agent 安全不应只靠“更保守的 prompt”。SRE calibration 论文显示，在 degraded/corrupted input 下是否行动需要结构化评测和架构机制。
2. 数据选择正在从“选高 loss 样本”走向“选会改变长期决策的关键反馈”。有限 reward label、ESLM、VERDICT 都指向同一件事：训练预算应投到影响策略、恢复和模型排序的样本上。
3. Search Agent 训练可以借鉴 preference optimization，但 preference pair 应保留“意图一致性/证据一致性”约束，否则容易变成只追逐单一相关性指标。
4. 行为诊断比最终分数更重要。MORL diagnostics 与 SRE calibration 都提醒我们：同样成功率的 agent 可能有完全不同风险轮廓。

## 潜在研究 idea

- 构造 Agent Operational Calibration Bench：同一工具任务生成 clean、missing、corrupted、contradictory observations，评测 act/inspect/escalate 的校准曲线。
- 做 reward-label selection for tool-use trajectories：优先标注失败恢复点、工具返回冲突点、计划分叉点，而不是平均标注完整轨迹。
- 把 HPPO 风格的 hybrid-policy candidate pool 用到搜索轨迹：混合规则检索、当前 Search Agent 查询改写和人工高质量轨迹，再用引用覆盖率/可信度/答案可验证性做 preference。
- 为 Code Agent 建立 decision-preserving regression suite：用 VERDICT 思路筛出最能保持模型排序和训练配方差异的小型任务集合。
