## 今日重点趋势

1. Agent 评测正在从“最终答案是否正确”转向“过程是否可约束、可审计、可复现”。XFlow、ProcGrep、MetaSyn 都在不同层面把过程拆开：协议状态、行为指纹、检索-筛选-综合阶段指标。
2. Search Agent 的核心问题不再只是召回更多文本，而是让检索行为适配环境并抵抗外部内容污染。SearchGEO 关注网页操纵后的背书风险，Environment-aware IR 和 RL-Index 分别把学习放在 query 侧与 index 侧。
3. RAG 系统开始显式管理证据预算。SCAR 处理 chunk 边界，MAGE-RAG 用证据图组织长文档多模态信息，二者都适合迁移到长上下文 agent 的上下文压缩与证据选择模块。
4. Code Agent 的价值正在从生成代码扩展到运行、审计、工具发现和验证。Rapid Reviews 用 coding agent 跑工具，ProcGrep 看轨迹差异，VerIbmc 这类工作虽未进 Top 10，但提示“验证器反馈 + LLM refinement”是值得关注的闭环。

## 潜在研究 idea

1. 为工具使用轨迹设计类似 ProcGrep 的可压缩行为词表，再用 XFlow 式协议状态约束训练数据，形成“轨迹审计 + 协议执行”的 agent post-training 数据管线。
2. 把 SearchGEO 的网页操纵攻击与 RL-Index 的索引侧 rationale 结合，训练检索 agent 对恶意证据、SEO 文本和伪引用做拒绝或降权。
3. 用 MetaSyn 的阶段化指标评估科研 agent：检索召回、筛选准确、证据引用、统计/综合结论分别计分，避免单一 end-to-end 分数掩盖瓶颈。
4. 把 SCAR/MAGE-RAG 的证据预算策略迁移到代码仓库阅读：文件、函数、调用边、测试失败日志都作为不同粒度证据节点，在线展开而不是一次性塞入上下文。
