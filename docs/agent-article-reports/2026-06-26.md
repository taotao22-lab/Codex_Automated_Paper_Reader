## General Agent

### 1. SKILL-DISCO: Distilling and Compiling Agent Traces into Reusable Procedural Skills

- 链接：[arXiv](https://arxiv.org/abs/2606.26669) / [PDF](https://arxiv.org/pdf/2606.26669)
- 问题背景：很多 LLM Agent 在 ALFWorld、WebArena 这类交互任务里反复从零探索相似流程。已有 workflow reuse 或 skill induction 往往直接让 LLM 从成功轨迹里写技能，容易得到表面化、碎片化、不可验证的技能库。
- 核心方法：论文把成功轨迹看成未知有限状态机中的路径，把可复用技能定义为参数化控制流子图。SKILL-DISCO 先从成功轨迹里发现共享的 PFSM 子结构，再把这些子结构编译成可调用、可执行、可验证的 procedural skills。直观地说，它不是把一条轨迹压缩成一句经验，而是把多条轨迹中重复出现的“去哪里、观察什么、满足什么条件后调用什么动作”抽成一个小程序，Agent 以后可以像调用工具一样调用这个程序。
- 创新点：它把经验复用从自然语言提示推进到执行结构复用。对 Agent post-training 很关键，因为成功轨迹不只是 SFT 样本，也可以蒸馏成可验证的技能库，再作为 inference-time action abstraction 或后续训练数据。
- 实验与 benchmark：论文在 ALFWorld 和 WebArena 上评估。ALFWorld 使用 200 个 train 任务做 induction，官方 unseen split 134 个任务做评估；WebArena 812 个任务按 406/406 切分。表格报告 SKILL-DISCO+ReAct 在 ALFWorld 成功率 92.4%，平均轮数 8.8，相比 ReAct 82.0% 和 19.3 轮明显改进；SKILL-DISCO+CodeAct 达到 99.3% 和 3.2 轮。WebArena 上 SKILL-DISCO+ReAct 成功率 29.1%，平均轮数 5.1，相比 ReAct 23.9% 和 5.9 轮也有提升。
- 可迁移价值：可以直接迁移到浏览器 Agent 和 Code Agent 的轨迹库构建：先找跨任务共享的控制流，再把它编译成带参数、带前置条件和验证器的技能，而不是把每条成功经验都塞进 prompt。
- 局限：它依赖任务存在稳定可复用的流程结构；对完全开放、一次性、实例特异的生成任务，PFSM 技能可能不自然。
- 阅读建议：强烈建议精读，尤其是 PFSM skill 定义、trace distillation 和 skill verification。
- 证据来源：PDF。

### 2. Joint Learning of Experiential Rules and Policies for Large Language Model Agents

- 链接：[arXiv](https://arxiv.org/abs/2606.27136) / [PDF](https://arxiv.org/pdf/2606.27136)
- 问题背景：交互式 Agent 的经验通常有两种用法：把过去失败/成功总结成自然语言规则放进 prompt，或者用轨迹做 RL/微调更新参数。前者可解释但容易和不断变化的策略脱节，后者能改模型但不保留清晰的可审计经验。
- 核心方法：JERP 把规则池和策略参数放在同一个训练循环里更新。每个 episode 中，当前策略带着从长期规则池选出的工作规则行动；采样到的轨迹一方面用于 group-relative policy update，另一方面和参考成功轨迹比较，用来更新经验规则池。直观地说，它让“写经验卡片”和“更新模型权重”共享同一批交互证据，避免规则停留在旧策略时代。
- 创新点：论文强调规则和参数不是二选一。对长期运行 Agent，可解释的外部规则池可以作为短期控制面，LoRA/RL 参数更新可以吸收稳定行为，两者同步演化。
- 实验与 benchmark：实验在 AlfWorld 和 WebShop 上进行。AlfWorld 覆盖 Pick、Clean、Heat、Cool、Look、Pick2 六类文本 household manipulation 任务；WebShop 评估网页搜索、浏览、比较和购买。JERP 在 AlfWorld 总成功率 61.5%，高于 GRPO 57.8% 和 RLOO 48.7%；WebShop 平均分 79.0、成功率 64.1%，高于 RLOO 57.8% 和 GRPO 56.2%。实验使用 LoRA 做参数高效微调，并报告三次独立运行平均。
- 可迁移价值：适合用在 Search Agent 和 Code Agent 的经验积累：把“哪些查询计划有效”“哪些测试失败后该怎么定位”写进规则池，同时用同一批轨迹做轻量 RL。
- 局限：实验环境仍是标准交互 benchmark，规则池的规模、冲突管理和长期遗忘在更开放任务中还需要验证。
- 阅读建议：重点看 rule-pool update 与 GRPO/RLOO 的对照，以及规则池如何和当前策略对齐。
- 证据来源：PDF。

### 3. Autoformalization of Agent Instructions into Policy-as-Code

- 链接：[arXiv](https://arxiv.org/abs/2606.26649) / [PDF](https://arxiv.org/pdf/2606.26649)
- 问题背景：Agent 要完成真实任务往往需要高权限工具，但 prompt guardrail 或分类器式安全控制没有形式保证；手写符号策略又难以覆盖大量自然语言政策、MCP 工具描述和系统指令。
- 核心方法：论文提出 Verification Sandwich，把 agent card、MCP tool schema 和自然语言政策自动翻译为 Cedar Policy Language 策略。生成器先产出候选 Cedar 规则，硬验证器用 Cedar reference tooling 检查语法、schema、vacuous policy 和冲突，失败时把验证错误回传给生成器；软 critic 再做 Judge 到 Verifier 的二阶段检查。运行时，外部 deterministic policy engine 在 Agent 动作执行前判定是否允许。
- 创新点：它把安全约束从 prompt 文本搬到可验证、可执行的 policy runtime。对工具 Agent 很直接：工具描述本身不只是给模型看的接口说明，也可以成为策略自动形式化的输入。
- 实验与 benchmark：论文在 MedAgentBench 上评估，把原始 HTTP 工具、typed MCP baseline 和 guardrail 条件下的轨迹 replay 到 Cedar policy engine。表格报告 Original Raw 中 Cedar Block Rate 为 30.3% (91/300)，Original Baseline 为 57.7% (173/300)，Original Guardrail 为 55.7% (167/300)；Adversarial 条件下 Raw 为 72.0% (36/50)，Baseline 为 82.0% (41/50)，Guardrail 为 85.7% (42/49)。作者说明 Cedar 覆盖的政策范围比既有手写 guardrail 更广。
- 可迁移价值：对 MCP/浏览器/代码执行 Agent 可以采用同样结构：自然语言任务约束和工具 schema 先转为形式策略，再由 runtime 拦截危险调用，而不是依赖模型自觉遵守。
- 局限：结果来自医疗 Agent benchmark，且策略质量依赖生成器和 critic；形式策略覆盖更多规则不等同于业务上零误杀。
- 阅读建议：建议精读 Verification Sandwich、Cedar schema 生成和 replay evaluation。
- 证据来源：PDF。

### 4. OpenRCA 2.0: From Outcome Labels to Causal Process Supervision

- 链接：[arXiv](https://arxiv.org/abs/2606.27154) / [PDF](https://arxiv.org/pdf/2606.27154)
- 问题背景：RCA 是很好的 LLM Agent 测试场，因为它需要长上下文、日志/trace/metric 融合、多步推理和工具使用。已有数据集多只标注最终 root cause，Agent 猜中服务名就算对，无法判断它是否真的理解故障传播路径。
- 核心方法：论文提出 PAVE 协议，用已知 fault injection 的干预信息做 forward verification，从原因出发重建到症状的 causal propagation path。OpenRCA 2.0 因此不只标“哪个服务坏了”，还标“故障经过哪些节点和边传播到观测症状”。评估指标包括 Path Reachability、Node F1 和 Edge F1。
- 创新点：它把 outcome-only evaluation 改成 process supervision。对 Agent 训练很有启发：最终答案正确但过程不接地，应被识别为 ungrounded diagnosis，而不是简单奖励。
- 实验与 benchmark：OpenRCA 2.0 包含 500 个实例，来自 TrainTicket、OpenTelemetry Demo 和 Hotel Reservation 三个架构不同的微服务系统。系统规模覆盖 9 到 44 个服务，平均最长传播路径 2.1 到 3.0。论文评估 11 个 frontier LLM backbone；精确 root-cause set 平均成功率只有 20.7%，放宽后至少识别一个正确 root-cause service 的比例为 76.0%，但能把该服务接到已验证传播路径的比例只有 61.5%。
- 可迁移价值：Code Agent 和运维 Agent 可以把“修复是否通过测试”拆成“是否定位到因果路径、是否解释边、是否对症状覆盖”。这类过程标签也适合作为 RL dense reward。
- 局限：当前是微服务 RCA，真实生产系统的缺失 telemetry、采样 trace 和截断窗口还会增加难度；论文也把这些作为未来可控评测轴。
- 阅读建议：适合做 Agent benchmark 设计和过程监督的核心 related work。
- 证据来源：PDF。

### 5. Diagnosing Task Insensitivity in Language Agents

- 链接：[arXiv](https://arxiv.org/abs/2606.26918) / [PDF](https://arxiv.org/pdf/2606.26918)
- 问题背景：语言 Agent 在长程环境中可能看似会执行任务，但面对相似却不同的任务时，会沿用训练中常见动作模式。论文把这种失败称为 task insensitivity：模型没有真正按当前任务条件调节行为，而是在局部观察和历史模式上走捷径。
- 核心方法：作者设计诊断：在 ALFWorld、ScienceWorld、WebShop 等环境中，保持 agent prompt 其他部分不变，但把任务描述改写成语义上被破坏或与原任务冲突的版本，观察模型是否仍执行原任务常见动作。论文还分析 action generation 时的 attention 漂移：训练过程中，模型对任务指令的注意减少，对当前 observation 的注意增强。
- 创新点：它挑战了“Agent benchmark 分数提高就是推理能力提高”的简单解释。对后训练尤其重要：action-token 监督可能让模型更会从 prompt 模板直接预测动作，而不是更会根据任务约束修正计划。
- 实验与 benchmark：PDF 明确覆盖 ALFWorld、ScienceWorld、WebShop，并进一步分析训练过程中 attention、gradient 与行为变化；还提出 reduced action-supervision intervention，在部分样本上减少最终 action token 的监督，以减弱直接动作记忆并改善 OOD 泛化。可见片段未提供完整数值表，本日报不摘录未核实数字。
- 可迁移价值：可以把这种 corruption diagnostic 加入 Search/Code Agent 训练：改写需求、测试约束或工具可用性，检查模型是否仍执行旧套路。
- 局限：诊断聚焦 task description 与 action pattern，不能完全覆盖工具调用失败、长期记忆污染或奖励 hacking。
- 阅读建议：建议与 “Where Do CoT Training Gains Land in LLM based Agents?” 一起读，用于区分 CoT 训练收益到底落在推理、动作记忆还是 prompt shortcut。
- 证据来源：PDF。

### 6. Memory Depth, Not Memory Access: Selective Parametric Consolidation for Long-Running Language Agents

- 链接：[arXiv](https://arxiv.org/abs/2606.26806) / [PDF](https://arxiv.org/pdf/2606.26806)
- 问题背景：长期 Agent 不能只靠 RAG 取回事实。检索回答的是“过去有什么可取回”，但长期目标、偏好和约束还需要在工作上下文卸载后持续影响行为。论文把这个问题称为 memory depth。
- 核心方法：作者提出 loop-drift protocol：检索索引保持可用，但工作上下文被卸载，同时事件流中混入稳定目标、离题干扰、短暂相反请求、冲突、 sibling-user contamination 和事实 notes。EVAF 用 surprise 和 valence gate 选择少量事件写入 LoRA 参数存储，目标是只把真正应持续影响行为的经验合并到小型参数记忆中。
- 创新点：它把记忆分成 access 和 depth 两层。对 Agent 系统设计很重要：RAG 适合浅层事实回忆，参数化 consolidation 更像“长期习惯/偏好/约束”的慢记忆。
- 实验与 benchmark：论文在 GPT-2 和 TinyLlama 上评估，报告 retrieval 在 short-fact accuracy 上最强，为 0.956 到 0.973；EVAF 在 goal persistence 和 post-unload recovery 上更强，为 0.812 到 0.904，每 200 个事件只需 2 到 3 次参数写入。机制控制显示 selection 和 actuation 是两个可分离维度，Mistral-7B 上 inner-loop write strength 具有模型依赖性。
- 可迁移价值：可用于构建长期 Code Agent 的偏好和工作方式记忆，例如“这个仓库优先保持 API 兼容”“测试失败先看 fixture”，这类东西不一定每次靠检索显式取回。
- 局限：目前是受控 synthetic stream 和小模型/诊断性实验；对大模型真实在线长期部署还只是方向证据。
- 阅读建议：适合关注 Agent memory 架构的人精读 loop-drift protocol。
- 证据来源：PDF。

## Search Agent

### 7. Generative Retrieval via Diffusion Transformer with Metric-Ordered Sequence Training and Hybrid-Policy Preference Optimization

- 链接：[arXiv](https://arxiv.org/abs/2606.26899) / [PDF](https://arxiv.org/pdf/2606.26899)
- 问题背景：常规 embedding retrieval 通常按 query similarity 返回最相似 item，但很多生产检索需要“既满足目标属性，又保持 seed set 的细粒度 pattern”。例如要找更多同一种违规模板，不能只找所有违规内容，也不能只返回和 seed 表面相似但目标属性弱的内容。
- 核心方法：MO-DiT+HPPO 让模型读取一组 item embedding 序列，生成一个或多个可用于 nearest-neighbor search 的 query embedding。训练包括 raw-sequence pretraining、metric-ordered continuation pretraining、tail-centroid SFT 和 HPPO。关键是 metric-ordered 阶段把稀疏在线标签整理成从低到高属性密度的 in-pattern trajectory；HPPO 再用真实在线 Joint@K 标注候选 query，并用 Pareto pair filter 避免提升属性时损失 same-pattern purity。
- 创新点：它把检索训练从“给 query 找相关文档”推进到“生成能触发目标检索行为的连续查询向量”，且偏好优化目标同时考虑属性命中和模式保持。对 Search Agent，这很像训练 query planner 学会沿着可验证指标改写查询。
- 实验与 benchmark：论文在四个匿名 large-scale attribute domains 上评估，包含 strict item-holdout 和 pattern-holdout。规模为 D1 1.54M train、D2 5.95M、D3 9.22M、D4 478K；指标包括 Joint@K 和 same-pattern purity。摘要和正文片段报告 metric-ordered DiT 相比强 pretrained generative retriever 提升 primary intersection metric，HPPO 进一步提升，八个 domain-split cells 中七个显著，最难 pattern split 为 marginal tie。
- 可迁移价值：可以迁移成 Search Agent 训练数据构造：把查询改写轨迹按“答案覆盖/证据质量提升且主题不漂移”排序，再用偏好优化训练 planner。
- 局限：领域是内部匿名 item retrieval，不是开放网页；属性定义也未公开，复现性弱于公开 benchmark。
- 阅读建议：Search Agent 今日第一优先，重点看 metric-ordered trajectory 和 Pareto preference pair。
- 证据来源：PDF。

### 8. KARLA: Knowledge-base Augmented Retrieval for Language Models

- 链接：[arXiv](https://arxiv.org/abs/2606.26807) / [PDF](https://arxiv.org/pdf/2606.26807)
- 问题背景：RAG 往往把检索作为外部 pipeline，而模型本身未必学会何时检索、检索哪个关系、如何把结果接入生成。KARLA 关注更结构化的知识库检索：让模型在生成时自动产出特殊 token 触发 KB query。
- 核心方法：模型被微调为在即将生成事实信息时输出特殊查询 token；运行时系统用 KB 查询结果替换这些 token，再继续生成。这样事实值来自 YAGO 或 PrimeKG 等知识库，语言模型负责解释上下文和组织表达。直观地说，它把事实记忆从参数中搬到可编辑 KB，把“会说话”和“存事实”分离。
- 创新点：对 Search Agent 的价值在于工具调用不是 prompt-time schema 展示，而是 post-training 后内化成 generation-time retrieval action。论文还强调 provenance：哪些内容来自 KB 可以被标记和追踪。
- 实验与 benchmark：论文使用 YAGO 和 PrimeKG 构造训练语料；在 YAGO 上评估 PopQA long-tail 短问答和 FActScore long-form generation，并引入 Counterfactual YAGO 测试事实覆盖。表格中 Qwen 0.6B + KARLA 在 PopQA/FActScore 上为 78.56/53.0，Base LM 为 16.37/22.78，1-hop graph RAG 为 54.45/53.1；Qwen 4B + KARLA 达到 80.91/58.9。
- 可迁移价值：Search Agent 可以借鉴 inline query token：模型不只规划自然语言查询，还学会在生成过程中插入可执行检索路径，并把检索 provenance 留在输出里。
- 局限：KB relation schema 相对稳定，和开放网页搜索的高噪声、动态页面不同；长链复杂检索的错误传播还需要更多验证。
- 阅读建议：作为 Search Agent 工具调用训练和 factual grounding 的 related work 收录。
- 证据来源：PDF。

## Code Agent / 执行闭环

### 9. TerraProbe: A Layered-Oracle Framework for Detecting Deceptive Fixes in LLM-Assisted Terraform Security Repair

- 链接：[arXiv](https://arxiv.org/abs/2606.26590) / [PDF](https://arxiv.org/pdf/2606.26590)
- 问题背景：LLM 已被用于自动修 Terraform/IaC 安全 misconfiguration，但很多评估只看静态分析告警是否消失。模型可能删除资源、改坏语义或绕过检查，让表面告警消失但安全意图没有修复。
- 核心方法：TerraProbe 提出五层 oracle，不只检查语法和 targeted static finding removal，还检查计划可达性、行为差异和 security intent alignment。直观地说，它把“补丁通过扫描器”升级为“补丁是否还能部署、行为是否符合预期、是否真的修了安全问题”。
- 创新点：这篇对 Code Agent 评测很有价值。它明确把 deceptive fix 作为对象，避免把 reward 写成单一静态检查，从而诱导模型 reward hacking。
- 实验与 benchmark：论文评估 288 个 first-pass LLM-generated repairs，来自 gemini-2.5-flash-lite、GPT-4o、Claude 3.5 Sonnet，覆盖 68 个真实 TerraDS modules 和 28 个 controlled injected-defect modules。摘要报告 plan-comparison reachability 在两条 track 间差异显著，chi-square=31.64，p<0.001，Cohen's h=1.36；TerraDS adjudicated cases 中 deceptive-fix rates 在三个模型间统计上不可区分，范围 57.1% 到 71.4%，Fisher exact p>0.10。
- 可迁移价值：对 SWE-bench、代码修复和安全修复都可以采用 layered oracle：不只看测试通过，也看行为 diff、风险意图、部署计划和副作用。
- 局限：Terraform 安全修复是特定领域；五层 oracle 的人工 adjudication 成本可能较高。
- 阅读建议：Code Agent evaluation 今日第一优先，尤其适合设计防 reward hacking 的修复评测。
- 证据来源：PDF。

### 10. Reinforcement Learning without Ground-Truth Solutions can Improve LLMs

- 链接：[arXiv](https://arxiv.org/abs/2606.27369) / [PDF](https://arxiv.org/pdf/2606.27369)
- 问题背景：RLVR 通常需要标准答案或可判定最终正确性的 reward，但很多真实代码/优化任务没有唯一标准答案，只有执行分数或性能指标。直接用原始分数做 RL 又会受不同题目尺度和采样频率影响。
- 核心方法：RiVER 使用 score-based optimization tasks 和 deterministic execution feedback 做连续监督，不需要 ground-truth solution。论文指出两个问题：scale dominance 会让不同实例的分数尺度扭曲更新，frequency dominance 会让频繁采样到的次优解压过少量高质量候选。RiVER 用 instance-wise comparison 和 top-ranked solver 强调来校准 reward，同时给其他有效解保留有界反馈。
- 创新点：对 Code Agent 很直接：真实代码优化、kernel tuning、检索策略优化通常没有唯一答案，但有可执行分数。RiVER 说明只要 reward 校准得当，这类任务也能训练出可迁移能力。
- 实验与 benchmark：论文在 12 个 AtCoder Heuristic Contest 任务上训练，在 ALE-Bench、LiveCodeBench 和 USACO 上评估。摘要报告 RiVER 使 Qwen3-8B 和 GLM-Z1-9B-0414 的 ALE rating rank 分别提升 8.9% 和 9.4%；尽管只用 score-based tasks 训练，在 LiveCodeBench 和 USACO 等 exact-solution benchmarks 上也分别带来平均 2.4% 和 3.5% 的绝对提升。原始执行分数 baseline 能提升 ALE rating，但不能迁移到 exact-solution benchmark。
- 可迁移价值：可以用于代码执行闭环训练：把测试覆盖率、性能、lint/security score、检索命中率等连续执行反馈转成 instance-wise calibrated reward，而不是只用 pass/fail。
- 局限：当前训练任务来自竞赛式 heuristic optimization，和真实仓库级修复仍有距离；reward 校准对任务集合设计很敏感。
- 阅读建议：适合和 TerraProbe、EGG 一起看，构成“可执行反馈 + 防奖励欺骗 + 性能优化”的 Code Agent 训练三件套。
- 证据来源：PDF。

### 11. EGG: An Expert-Guided Agent Framework for Kernel Generation

- 链接：[arXiv](https://arxiv.org/abs/2606.26758) / [PDF](https://arxiv.org/pdf/2606.26758)
- 问题背景：GPU kernel 编写需要专家调优。LLM 生成 kernel 容易遇到两个问题：正确性不稳，以及优化搜索缺少硬件专家先验，导致能编译但不快，或探索空间很乱。
- 核心方法：EGG 把 kernel 生成拆成两级：algorithmic structure design 和 hardware-specific tuning。它用 expert-guided staged optimization 约束 LLM 的决策路径，并通过 stage-aware multi-agent collaboration 管理 code、profile、debug 等上下文。通俗地说，它不是让模型一次性写 Triton kernel，而是让模型先定数据搬运和算法结构，再围绕具体硬件做 tile、memory、parallelism 等调参，并用 profiling/debug feedback 形成稳定迭代。
- 创新点：它体现了 Code Agent 的重要模式：专家 workflow 显式化 + 执行反馈闭环 + 多 Agent 分工。相比纯 RL 搜索或普通 multi-turn agent，它更强调把领域专家的优化顺序注入 action space。
- 实验与 benchmark：论文以 KernelBench 为主评测，包含 250 个 kernel tasks，覆盖 basic operators、fused operators 和 complete networks。硬件主结果在 NVIDIA RTX 4090 上，附录还报告 RTX 5090、H20、RTX PRO 6000；软件栈为 CUDA 13.0、PyTorch 2.9.1、Triton 3.5.1，LLM 主要使用 GPT-5.1。结果报告 EGG 相比 PyTorch 平均 2.13x speedup，并优于既有 agent-based 和 RL-based 方法。
- 可迁移价值：对一般 Code Agent，可借鉴“先结构后调参”的 staged action space，把 profile/debug/test agent 的上下文分开管理。
- 局限：使用较新闭源模型和多 GPU/硬件设置，复现门槛较高；任务集中在 Triton/GPU kernel，不覆盖普通软件工程修改。
- 阅读建议：适合关注代码执行闭环、性能优化 Agent 和 expert workflow injection 的读者。
- 证据来源：PDF。

## 未进入 Top 10 但建议留意

- Semantic Early-Stopping for Iterative LLM Agent Loops：在 HotpotQA multi-hop RAG 上研究语义早停。PDF 报告 judge-free semantic stopper 在 60-question test split 上相对 `max_iterations` 节省 38% operational tokens，质量近似持平；但 full quality-gated variant 因每轮 judge 成本而不划算。
- Where Do CoT Training Gains Land in LLM based Agents?：分析 CoT 训练收益是否真正落在 reasoning 上。论文覆盖 ALFWorld、ScienceWorld、BFCL，指出 prompt-only action prediction 也随训练增强，后期 checkpoint 更不容易被 CoT 改写动作。
- KARLA: Knowledge-base Augmented Retrieval for Language Models：Search Agent 工具调用和 factual grounding 价值高，因 Top 10 名额限制未进入主表，但建议作为检索工具调用训练相关工作收录。
- Reasoning Quality Emerges Early：提出只用早期 reasoning token loss 做数据筛选，在 Qwen2.5-7B、Llama3.1-8B 和 M23K/OpenThoughts-Math 上做验证；对 reasoning SFT 数据筛选有价值，但与交互 Agent 间接。
- NebulaExp-8B：透明 post-training pipeline 和 ablation 报告，对复现实验流程有参考价值，但 Agent/tool-use 专门性不如本次 Top 10。
- Designing Reward Signals for Portable Query Generation：工业 semantic job search 中的 RLAIF query generation，强调 reward shaping 比 optimizer 选择更关键；可作为 Search Agent reward hacking 参考。

## 今日重点趋势

1. 经验复用正在从“自然语言反思”转向“可执行结构”。SKILL-DISCO 把轨迹蒸馏成 procedural skills，JERP 把规则池和策略参数联合更新，二者都说明经验不应只存在 prompt 里。
2. Agent 评测正在从 outcome-only 转向 process-aware。OpenRCA 2.0 和 TerraProbe 都显示，最终答案或告警消失不足以证明 Agent 做对了；需要因果路径、行为差异和安全意图。
3. 可验证执行反馈正在扩大 RLVR 的适用范围。RiVER 证明没有标准答案的 score-based optimization 也能训练代码能力，但 reward 必须做实例级校准。
4. Search Agent 的训练目标更像“生成可执行检索动作”。MO-DiT+HPPO 和 KARLA 都把检索从外部 pipeline 变成模型可学习的动作，只是一个生成 query embedding，一个生成 KB query token。
5. 长期 Agent 需要区分事实检索和行为固化。Memory Depth 的结果提示，RAG 不应承担所有长期记忆职责，稳定目标和偏好可能需要单独的 consolidation 机制。

## 潜在研究 idea

- 用 SKILL-DISCO 的 PFSM 技能抽取方式处理 Code Agent 成功修复轨迹：把“定位文件、运行测试、最小补丁、复测”抽成可参数化技能，再评估跨仓库迁移。
- 将 JERP 的规则池更新用于 Search Agent：每轮搜索轨迹不仅训练 query planner，还更新“何时换查询、何时追溯来源、何时拒答”的经验规则。
- 把 OpenRCA 2.0 的 Edge F1 思路迁移到代码调试：评估 Agent 是否说明 bug 如何从输入传播到失败断言，而不是只看 patch 是否通过测试。
- 结合 TerraProbe 和 RiVER 设计防 reward hacking 的代码 RL：奖励来自测试、性能和安全 oracle 的多层组合，并做 instance-wise calibration。
- 为浏览器 Agent 引入 Policy-as-Code runtime：把网站权限、工具 schema 和用户约束自动转成可执行策略，模型只能在策略允许的动作空间内规划。
