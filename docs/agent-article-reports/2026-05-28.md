今天的高价值论文集中在三个方向：一是把 Agent 运行时本身做成可训练、可验证、可治理的程序对象；二是把评测从静态 benchmark 推向真实 GUI、长流程和生产运行时；三是把检索、记忆、规格综合等模块做成可迁移到 Agent post-training 的中间能力层。

## General Agent

- **LACUNA**：把 agent action 变成带类型约束的“递归程序洞”，核心价值是把 ReAct、sub-agent、skills、并行分解都统一到类型检查过的控制流里，适合拿来思考“安全的 code-as-action runtime”。
- **CaveAgent**：把持久 Python runtime 提升为 Agent 的主工作区，而不是只让模型输出文本或 JSON function call。对长程任务、状态保持、可验证奖励信号都很关键。
- **Tool Forge**：重点不在“让模型会调工具”，而在“把工具生产、验证、路由、审计做成一条治理链路”。对企业级 agent execution 非常有工程启发。
- **AndroidDaily / AgenticVBench**：两篇都在说明真实 GUI/多模态工作流下，当前 agent 与 human expert 仍有明显差距，而且 harness 设计本身会显著改变结果。

## Search Agent

- **Search for Coverage**：不是继续卷相关性，而是显式学习“覆盖多个子问题”的 long-form retrieval，这对搜索代理、RAG 轨迹监督、答案验证都更直接。
- **MGRetrieval**：把长期记忆检索做成“memory-guided reflective retrieval”，避免只靠 LLM 临时生成路径，适合迁移到长上下文 agent 和记忆压缩系统。
- 值得关注但未入 Top 10：**Do Agents Need Semantic Metadata?** 对“开放网页检索 vs. 结构化语义元数据”给出了很实用的对比结论，适合作为 agentic retrieval 的相关工作。

## Code Agent

- **RAMP**：把长流程软件工程 agent 放进真实 runtime 中评测，而不是只看静态 benchmark，直接揭示跨阶段失败传播与资源浪费。
- **Spec-Agent**：把静态分析、heap tracing、fuzz harness、counterexample-guided refinement 组合成仓库级规格综合闭环，非常贴近“代码执行反馈驱动修复”。
- **GUI Agents for Continual Game Generation**：把 code agent 和 GUI playtester 绑定成持续回路，说明“可运行”不等于“可交互”，这是闭环训练的重要补充视角。

## Top 10 Papers

### 1. LACUNA: Safe Agents as Recursive Program Holes
- 评分：4.735
- 论文页：https://arxiv.org/html/2605.28617
- 核心方法：把 `agent[T](task)` 作为带静态类型约束的 primitive；模型写出的代码先通过类型检查，再允许执行。
- 迁移价值：非常适合 General Agent 的 runtime 设计，尤其是把 ReAct loop、sub-agent、skill 调用、多模型规划统一为普通控制流。
- 可转化实验/模块：可以做一个“typed tool-use sandbox”，比较静态拒绝率、重试成本、提示注入防御成功率。
- 局限：当前实现落在 Scala 3 生态，真实多工具环境中的开发者体验与跨语言落地成本还需要验证。
- 证据来源：阅读 arXiv HTML 的 `Introduction`、`Typed Holes as Agents`、`Evaluation`、`Discussion and Future Work`。

### 2. CaveAgent: Transforming LLMs into Stateful Runtime Operators
- 评分：4.710
- 论文页：https://arxiv.org/html/2601.01569
- 核心方法：双流架构把语义推理流与持久 runtime 流解耦，让 Python 对象、数据库连接、DataFrame 等跨轮次常驻。
- 迁移价值：对 tool-use reasoning、外部记忆、RLVR、token 效率优化都很强，尤其适合做“runtime state 作为训练信号”的后训练研究。
- 可转化实验/模块：把 persistent object state 接到现有代码 agent 中，比较 function calling、CodeAct、runtime-first 三种协议的成功率与 token 开销。
- 局限：当前实验主要证明运行时范式有效，尚未完全展开对安全边界、隔离级别和更复杂副作用工具的分析。
- 证据来源：阅读 arXiv HTML `Introduction`、`Core Methodologies`、`Experiments`、`Conclusion`。

### 3. Tool Forge: A Validation-Carrying Toolchain for Governed Agentic Execution
- 评分：4.645
- 论文页：https://arxiv.org/html/2605.28000
- 核心方法：把 tool 视为带 intent、contract、implementation、tests、validation evidence、routing metadata 的“胶囊”，再通过 intent-scoped router 暴露给 agent。
- 迁移价值：直接对应“如何让 agent 工具链可治理、可审计、可复用”。对 MCP/tool catalog、sandbox、policy enforcement 很有参考价值。
- 可转化实验/模块：可以复现 intent-scoped tool session，与“完整 schema 全量塞进上下文”的基线比较工具选择质量与上下文成本。
- 局限：论文把结果定义为初始 systems benchmark，不是 SOTA 证明；对更强对抗路由与跨系统评测还未覆盖。
- 证据来源：阅读 arXiv HTML `Introduction`、`Tool Forge Conceptual Framework`、`Experimental Protocol and Baselines`、`Conclusion`。

### 4. Benchmarks are Not Enough: RAMP for Runtime Assessing of Agentic Models in Production Systems
- 评分：4.615
- 论文页：https://arxiv.org/html/2605.27492
- 核心方法：构造生产化编译器工作流，把 agent 放进串行依赖、工具链交互、分阶段恢复和多维效用指标中评测。
- 迁移价值：对 Code Agent post-training 很重要，因为它强调“跨阶段失败传播”而不是单题对错，更接近真实仓库任务。
- 可转化实验/模块：把现有代码 agent 的 benchmark 扩成 staged runtime pipeline，并记录每阶段 completion、修复成功率、token/时间成本。
- 局限：当前任务域主要是 compiler construction，跨到 Web/ML/DevOps 场景时还需要新的环境设计。
- 证据来源：阅读 arXiv HTML `Introduction`、`Assessment Framework`、`Results and Analysis`、`Discussion on Limitations and Future Work`。

### 5. Agentic Separation Logic Specification Synthesis
- 评分：4.565
- 论文页：https://arxiv.org/html/2605.27531
- 核心方法：用静态分析、runtime heap tracing、fuzzing 与 counterexample-guided refinement 让 Spec-Agent 在大规模 C++ 仓库中综合规格。
- 迁移价值：这篇的重点不是“证明辅助”本身，而是把 repo-scale code understanding、验证反馈、candidate refinement 串成稳定闭环。
- 可转化实验/模块：可以把 fuzz-guided refinement 的模式借到 bug fixing 或 test synthesis 中，做执行反馈驱动的 agent 训练数据。
- 局限：目标任务聚焦规格综合，不直接覆盖通用 issue resolution；需要判断迁移到普通软件工程任务时的收益保真度。
- 证据来源：阅读 arXiv HTML `Introduction`、`Spec-Agent Design`、`Evaluation`、`Conclusion`。

### 6. AndroidDaily: A Verifiable Benchmark for Mobile GUI Agents on Real-World Closed-Source Applications
- 评分：4.495
- 论文页：https://arxiv.org/html/2605.27761
- 核心方法：在 94 个闭源 Android 应用上的 350 个真实任务中，用基于可观察外部准则的 GRADE 自动评测 GUI agent。
- 迁移价值：对 General Agent 尤其重要，因为它解决了“闭源真实应用没有内部状态可看时如何自动验证”的问题。
- 可转化实验/模块：可以把 guideline-grounded reviewer 思路移植到桌面/浏览器 agent，做 process-aware evaluator。
- 局限：当前最强模型也只有 62.0% 成功率，说明 benchmark 很硬；但也意味着训练和评测成本都会比较高。
- 证据来源：阅读 arXiv HTML `Introduction`、`Method`、`Evaluation of GUI Agents`、`Conclusion`。

### 7. GUI Agents for Continual Game Generation
- 评分：4.450
- 论文页：https://arxiv.org/html/2605.28258
- 核心方法：一方面提出 PlaytestArena，让 GUI agent 真正进入浏览器试玩并打分；另一方面提出 Play2Code，让 code agent 与 GUI playtester 在共享记忆中持续迭代。
- 迁移价值：它把“执行反馈”从编译/单测推进到交互行为层，适合启发 GUI agent、browser agent 和可玩性驱动的代码闭环训练。
- 可转化实验/模块：可以把 GUI playtester 替换成人机界面任务里的 reviewer agent，形成生成-执行-观察-修复闭环。
- 局限：任务域是浏览器游戏，环境信号和用户目标都较特殊；迁移到办公软件或通用网页任务需要重新设计 rubric。
- 证据来源：阅读 arXiv HTML `Introduction`、`Play2Code`、`Game Generation and Evaluation`、`Conclusions`。

### 8. AgenticVBench: Can AI Agents Complete Real-World Post-Production Tasks?
- 评分：4.435
- 论文页：https://arxiv.org/html/2605.27705
- 核心方法：基于 20 位行业专家真实工作流，构造 100 个视频后期 agent 任务，并混合程序验证与专家 rubric。
- 迁移价值：它说明多模态长流程 agent 的难点不在单步感知，而在任务编排、工具调用和 harness 设计；对评测集构造很有启发。
- 可转化实验/模块：可以借鉴其“专家工作流 -> 可验证任务族 -> 多 harness 对比”的方法，做 Search/Code Agent 的更真实 benchmark。
- 局限：任务域偏视频后期，直接迁移到通用 agent 需要抽象出更底层的 runtime failure taxonomy。
- 证据来源：阅读 arXiv HTML `Introduction`、`Task Family Framework`、`Experiments and Analysis`、`Discussion and Conclusion`。

### 9. Search for Coverage: Learning Coverage-Aware Retrieval with Augmented Sub-Question Answerability
- 评分：4.405
- 论文页：https://arxiv.org/html/2605.28522
- 核心方法：提出 CoveR，用 coverage-based contrastive/distillation objective 学会“多个文档合起来覆盖全部信息需求”，并用 SCOPE 数据集提供训练信号。
- 迁移价值：对 Search Agent 与 long-form RAG 非常直接，因为很多 agent 失败并不是找不到相关文档，而是漏掉关键子问题覆盖。
- 可转化实验/模块：可把 sub-question answerability 信号改造成搜索轨迹监督或 reward model 特征，用于 query planning / reranking。
- 局限：主要聚焦检索层，对生成端如何利用覆盖信号还需要配套实验。
- 证据来源：阅读 arXiv HTML `Introduction`、`Coverage-Aware Retrieval`、`Experimental Setup`、`Conclusion`。

### 10. MGRetrieval: Memory-Guided Reflective Retrieval for Long-Term Dialogue Agents
- 评分：4.235
- 论文页：https://arxiv.org/html/2605.27437
- 核心方法：不再只让 LLM 基于少量证据临时生成反思式检索路径，而是利用历史记忆的语义结构构造 retrieval path，并逐步压缩为充分上下文。
- 迁移价值：很适合作为长期记忆 agent 的中间模块，也适合和上下文压缩、episodic memory、self-reflection 联动。
- 可转化实验/模块：可以把 memory-guided path 用到代码仓库历史、网页浏览轨迹或长对话记忆中，观察是否能稳定减少无效检索。
- 局限：实验主要在对话与记忆数据集上，迁移到 Search Agent 或 Code Agent 还需要新任务验证。
- 证据来源：阅读 arXiv HTML `Introduction`、`Memory System`、`Experiment`、`Conclusions`。

## 今日重点趋势

1. **Agent runtime 正在从“提示词外壳”转向“可编程执行层”**。LACUNA、CaveAgent、Tool Forge 都在重写运行时边界。
2. **评测从静态 benchmark 转向真实交互/真实生产流**。RAMP、AndroidDaily、AgenticVBench、PlaytestArena 都强调 runtime observability。
3. **中间能力层比端到端大而全更值得投资源**。覆盖式检索、记忆检索、规格综合、工具治理都属于可复用模块。

## 潜在研究 Idea

1. 做一个 **typed runtime + persistent state + governed tools** 的统一 agent sandbox，把 LACUNA、CaveAgent、Tool Forge 三条线合并。
2. 用 **runtime-observable evaluator** 取代单纯终局打分：借鉴 RAMP/AndroidDaily，为 Search Agent 和 Code Agent 记录阶段性失败传播。
3. 以 **coverage / memory path / execution trace** 作为后训练监督信号，而不是只用最终答案正确性。
