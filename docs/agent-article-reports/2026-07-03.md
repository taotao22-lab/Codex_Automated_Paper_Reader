### Code Agent

## Top 10

### 1. PACE: A Proxy for Agentic Capability Evaluation

- 类别：General Agent
- 链接：[2607.02032](https://arxiv.org/abs/2607.02032)
- 作者：Song, Yueqi；Sutawika, Lintang；Liu, Jiarui；Tjuatja, Lindia
- 综合评分：4.9

**问题背景与核心方法**：PACE 解决的是一个很现实的工程问题：SWE-Bench、GAIA 这类 Agent benchmark 贵、慢、依赖复杂基础设施，日常模型迭代不可能每次完整跑。它把目标 Agent benchmark 的成绩看作可由一小组便宜“原子能力题”预测的变量，先从 19 个非 Agent benchmark 里构造候选实例池，再用 target-relevance local selection 和 globally informative global selection 选择子集，最后用回归映射到目标 Agent 分数。直观地说，它不是把 Agent 任务改写成选择题，而是用已知模型在便宜题上的表现，校准出能预测昂贵 Agent 任务的探针集合。

**创新点、实验与 benchmark**：创新点在于把 Agent 评测从“每次全量执行环境”变成“用代理题估计目标 benchmark”，并且还能分析不同 Agent benchmark 独有依赖的底层技能。实验覆盖 14 个模型、4 个 Agent benchmark（GAIA、SWE-Bench Verified、SWE-Bench Multimodal、SWT-Bench）和 19 个非 Agent benchmark；论文报告 PACE-Bench 的 LOOCV MAE 低于 4%、Spearman 超过 0.80、pairwise model-ranking accuracy 约 85%，成本低于完整 Agent 评测的 1%。局限是固定公开 proxy 可能被针对性优化，且校准模型只有 14 个。

**可迁移价值**：适合用来设计“训练中快速回归测试”：给 General/Code/Search Agent post-training 加一层低成本 proxy suite，先过滤明显退化，再周期性跑完整环境。

**证据来源**：PDF；阅读 abstract、introduction、method、experiments、conclusion、limitations。

### 2. Safety Testing LLM Agents at Scale: From Risk Discovery to Evidence-Grounded Verification

- 类别：General Agent / Safety
- 链接：[2607.01793](https://arxiv.org/abs/2607.01793)
- 作者：Feng, Yunhao；Lin, Ruixiao；Wen, Ming；He, Qinqin
- 综合评分：4.9

**问题背景与核心方法**：Vera 把 Agent 安全测试做成类似软件测试的可执行流水线，而不是只靠人工写 unsafe prompt。第一步从文献持续抽取风险、攻击方法和工具执行环境，形成 taxonomy；第二步把这些维度组合成可执行安全用例，每个用例包含安全目标、程序化初始状态和确定性 verifier；第三步在隔离沙箱中运行不同 Agent，由控制 Agent 根据运行时观察推进多轮交互，最终用环境状态和工具调用证据判断，而不是相信模型自述。

**创新点、实验与 benchmark**：创新点是把“风险发现、case 生成、执行、证据验证”闭成自强化循环，适合快速演化的 tool-use Agent。实验覆盖 OpenClaw、Hermes、Codex、Claude Code 四个生产 Agent 框架，释放 Vera-Bench：1600 个可执行安全用例、124 类风险、3 类执行设置；摘要报告多通道攻击平均成功率达到 93.9%。局限是具体风险 taxonomy 和 verifier 仍依赖持续维护，且强攻击成功率不等价于真实部署概率。

**可迁移价值**：对你的研究最有价值的是 case schema：安全目标 + 初始环境 + 可观测 predicate。这个格式同样可用于构造 tool-use RL 的负例、拒绝动作训练和执行轨迹审计。

**证据来源**：PDF；阅读 abstract、framework、evaluation/results。

### 3. ContextSniper: AntTrail's Token-Efficient Code Memory for Repository-Level Program Repair

- 类别：Code Agent
- 链接：[2607.01916](https://arxiv.org/abs/2607.01916)
- 作者：Luk, Chiwang；Najafi, Matin Mohammad；Jia, Zhifeng；Yang, Wei
- 综合评分：4.7

**问题背景与核心方法**：ContextSniper 针对仓库级修复中最常见的浪费：Agent 反复读整文件、扫宽搜索结果、塞长日志，真正有用的几行证据被噪声淹没。它作为 host coding agent 和 repo 之间的记忆层，先检索候选代码与运行时证据，再用混合检索信号排序，通过 intention-aware context gate 过滤长输出，返回紧凑 evidence packet，同时把可恢复的源上下文保留在 prompt 外。

**创新点、实验与 benchmark**：创新点不是又做一个 Agent，而是把“上下文摄取”变成可控、可复用、可评估的中间层。实验在 SWE-bench Lite 上接 OpenClaw 和 Claude Code，各 50 个任务条件；OpenClaw 总 token 降低 51.5%、成本降低 36.4%，Claude Code 总 token 降低 38.9%、估算成本降低 27.3%。代价是提交解决率略降：OpenClaw 26.0% 到 24.0%，Claude Code 32.0% 到 30.0%。局限是还需要更标准的 SWE-bench Verified 和更大样本验证。

**可迁移价值**：适合转化成 Code Agent 训练数据构造模块：把“搜索-读取-运行-过滤”的轨迹切成 evidence selection 决策，训练模型少带无关上下文。

**证据来源**：PDF；阅读 abstract、motivation、method、evaluation、limitations。

### 4. SkillCoach: Self-Evolving Rubrics for Evaluating and Enhancing Agentic Skill-Use

- 类别：General Agent / Skill
- 链接：[2607.01874](https://arxiv.org/abs/2607.01874)
- 作者：Zhu, Jiayin；Mao, Kelong；Guo, Yudong；He, Dengbo
- 综合评分：4.7

**问题背景与核心方法**：SkillCoach 关注 skill repository 时代的过程质量：Agent 最终成功不代表它选对 skill、按 skill 执行、正确组合 skill 或做了必要反思。它从真实 rollout 中演化 skill-grounded process rubric，沿 skill selection、skill following、skill composition、skill-grounded reflection 四个维度评价轨迹，并把外部 verifier 保持为单独 outcome signal。

**创新点、实验与 benchmark**：创新点是把 skill-use 的监督从“任务成功/失败”拆成可诊断过程维度，避免 accidental success 掩盖错误路径。摘要显示 evolved rubric 能提升评价质量、暴露 final accuracy 看不到的失败，并为筛选高质量训练轨迹提供比 outcome-only 更强的信号。局限是具体 benchmark、数值和泛化范围需要结合正文进一步复核；本次 PDF 抽取确认了方法目标和流程，但未逐表复核所有数值。

**可迁移价值**：这篇适合加入 Agent post-training 数据管线：先用 rubric 给轨迹打过程标签，再把“成功但过程差”的样本降权或剔除。

**证据来源**：PDF；阅读 abstract、method/evaluation snippets，部分表格未逐项人工核验。

### 5. Atomic Task Graph: A Unified Framework for Agentic Planning and Execution

- 类别：General Agent / Planning
- 链接：[2607.01942](https://arxiv.org/abs/2607.01942)
- 作者：Zhang, Yue；Chen, Sihan；Huang, Ziwen；Cui, Hanyun
- 综合评分：4.5

**问题背景与核心方法**：ATG 把多步任务的文本轨迹显式化为任务图。规划阶段递归分解成一系列 DAG，让子任务的输入输出依赖可追踪；执行阶段利用依赖关系并行执行无关分支；失败时根据图演化历史定位错误源，只修复受影响区域，保留已验证节点。它解决的是纯文本 plan 难以复用中间结果、难以局部回滚的问题。

**创新点、实验与 benchmark**：创新点是把“计划-执行-修复”的状态从 prompt 文本提升为可操作图结构，而且强调 training-free、跨任务可用。摘要称在三个交互 benchmark 上，用 7B-8B backbone 就能在成功率和执行效率上超过强 baseline。局限是摘要未列出 benchmark 名称和完整数值，本次报告不编造具体表格结果。

**可迁移价值**：可迁移到 Code/Search Agent 的轨迹表示：搜索 query、代码编辑、测试运行、验证结论都可以成为图节点，失败后只重跑受影响子图。

**证据来源**：PDF；阅读 abstract、introduction、method、experiments/conclusion，具体 benchmark 名称未从摘要确认。

### 6. ContextNest: Verifiable Context Governance for Autonomous AI Agent

- 类别：Search Agent / Context Governance
- 链接：[2607.02116](https://arxiv.org/abs/2607.02116)
- 作者：Sulpovar, Misha；Konsynski, Benn R.；Kanchwala, Qaish；Goodhart, Gabe
- 综合评分：4.5

**问题背景与核心方法**：这篇把 RAG 前面的“哪些知识允许被 Agent 使用” formalize 成 context governance。它不替代 retrieval，而是在检索前维护 typed Markdown 文档、元数据、确定性选择器、contextnest:// URI、SHA-256 hash-chained version history、graph checkpoints、MCP live source node 和 Agent consumption audit trace。直观地说，它让 Agent 输出可以追溯到当时被批准、完整、可归因的知识版本。

**创新点、实验与 benchmark**：创新点是把检索质量和治理保证分开：BM25/dense retrieval 解决相关性，ContextNest/ContextNext 解决版本身份、完整性、可重建性和 AI-eligibility。实验包括 stale-version attack 和 1,060 文档 retrieval-determinism；摘要报告 governed selection 在 stale-version attack 中 answer-quality pass rate 97%，BM25 为 93-90%，且约三分之一 input-token cost；确定性选择器和 BM25 重复查询 Jaccard 1.0，dense+HNSW 在 80% 查询上非确定，mean Jaccard 0.611。局限是它偏系统规范，效果依赖组织能否维护元数据和版本链。

**可迁移价值**：对 Search Agent 很关键：把引用可信度、时间点重建和 MCP source 消费日志纳入训练/评测，而不是只优化 recall。

**证据来源**：PDF；阅读 abstract、method/spec、experiments、limitations。

### 7. UA-ChatDev: Uncertainty-Aware Multi-Agent Collaboration for Reliable Software Development

- 类别：Code Agent
- 链接：[2607.02186](https://arxiv.org/abs/2607.02186)
- 作者：Ogunsusi, Temitayo Olamilekan；Qian, Lijun；Dong, Xishuang
- 综合评分：4.3

**问题背景与核心方法**：UA-ChatDev 继承 ChatDev 式多角色软件开发流程，但不再默认每个中间产物可靠。它用 token-level log probabilities 估计当前 agent response 的不确定性，并按阶段校准阈值；当不确定性超过阈值时，触发检索验证补充上下文，否则继续标准协作链。这样可以阻断早期需求分析、设计或代码阶段的幻觉继续传给下游 Agent。

**创新点、实验与 benchmark**：创新点是把 uncertainty quantification 放进 agent-to-agent communication，而不是只在最终答案或代码上做检查。实验使用 SRDD benchmark，以 completeness、executability、consistency、overall quality 评价；PDF 抽取显示论文声称相对单 Agent 和多 Agent baseline 均有提升，并通过消融与通信分析验证不确定性交互增强代码执行可靠性。局限是正文可抽取数值有限，本报告不摘未核实具体百分比。

**可迁移价值**：可迁移到多 Agent coding pipeline：把低置信阶段输出作为“必须检索/运行测试/请求验证”的触发器，而不是让所有消息同权流转。

**证据来源**：PDF；阅读 abstract、method workflow、evaluation metrics、experiments。

### 8. AgenticSTS: A Bounded-Memory Testbed for Long-Horizon LLM Agents

- 类别：General Agent / Long-Horizon Memory
- 链接：[2607.02255](https://arxiv.org/abs/2607.02255)
- 作者：Cheng, Xiangchen；Jiang, Yunwei；Sun, Jianwen；Li, Zizhen
- 综合评分：4.3

**问题背景与核心方法**：AgenticSTS 关注长程 Agent 的记忆契约：每一步决策不再简单追加完整 transcript，而是从 typed retrieval 组装 fresh user message，使 prompt 长度在任意运行长度下保持有界，并允许单独消融每一层记忆。测试环境选 Slay the Spire 2，因为它有封闭规则、随机性和数百步战术/战略决策。

**创新点、实验与 benchmark**：创新点在于“bounded-memory contract + 可复现实验资产”，不是单纯追求游戏胜率。论文释放 298 条完成轨迹、condition tags、冻结 memory/skill snapshots、prompt records 和分析脚本。摘要报告在 fixed-A0 消融中 no-store baseline 3/10 胜，加入 triggered strategic skills 6/10 胜，但 Fisher exact p≈0.37，作者明确把它视为方向性而非统计决定性结果。局限是样本规模小、游戏域特定。

**可迁移价值**：可用于研究长程 Agent 的记忆层设计：把 episodic、semantic、skill、reflection 分层，并要求每层都能被单独消融。

**证据来源**：PDF；阅读 abstract、method、experiments、limitations。

### 9. Evidence-State Rewards for Long-Context Reasoning

- 类别：Search Agent / Long-Context RL
- 链接：[2607.02073](https://arxiv.org/abs/2607.02073)
- 作者：Gao, Ya；Marttinen, Pekka
- 综合评分：4.3

**问题背景与核心方法**：Maven 把长上下文推理建模为“编辑证据记忆”的过程，而不是一次性抽证据或只奖励最终答案。模型可以 add、link、drop evidence；add 用边际增益和 hindsight contribution 给分，link 用证据协同给分，drop 用移除误导证据后答案支持度提升给分；这些奖励被分配到 GRPO 中对应的 action span。

**创新点、实验与 benchmark**：创新点是奖励中间证据状态转移，让模型学会导航、修正和合成证据。实验训练数据包含 9K 长上下文 QA，上下文 8K-64K tokens，来源包括 LongRLVR、HotpotQA、2WikiMultiHopQA 等；评测覆盖 LongBench v2、LongReason、RULER，并额外用 150 个带 gold evidence 的诊断样本看证据充分性和 distractor retention。摘要称 Maven 超过 outcome-only RL 和 evidence-identification baseline。局限是训练依赖 gold answers 和可验证答案，且 evidence interface 假设证据可被有界表示。

**可迁移价值**：对 Search Agent 训练很有启发：搜索轨迹不只奖励最终回答，还奖励“加入关键证据、连接证据、删除干扰证据”的状态改善。

**证据来源**：PDF；阅读 abstract、methodology、experiments、conclusion、limitations。

### 10. Coding-agents can replicate scientific machine learning papers

- 类别：Code Agent / Research Automation
- 链接：[2607.02134](https://arxiv.org/abs/2607.02134)
- 作者：Hans, Atharva；Bilionis, Ilias
- 综合评分：4.2

**问题背景与核心方法**：Paper-replication 把 coding agent 复现论文的过程做成 skill：先把论文计算声明拆成 target，重建方法、运行实验、把输出和 provenance 关联到原论文 claims，再在 replication report 中记录匹配证据，最后通过 validation checks 才算完成。它解决的是普通 prompt 容易丢进度、最终消息不等于真实证据的问题。

**创新点、实验与 benchmark**：创新点在于 completion gate 依赖工作区证据和检查，而不是 Agent 的自我报告。实验做了四篇 scientific ML 论文、每篇三次独立运行，共 12 个 workspace；摘要报告 12 个 workspace 全部通过 completion gate，158 个 recorded targets 都有 report coverage。但重复运行在 target 划分、数值 fidelity、耗时、被替换的中间执行次数和证据接受规则上差异明显。局限是论文数量小、没有无 skill 消融，所以它证明 workflow 产物形态，而不是严格量化 skill 带来的提升。

**可迁移价值**：这篇可作为 Code Agent 闭环训练模板：target ledger、execution provenance、claim matching、validation gate 都适合迁移到 issue repair 和实验复现任务。

**证据来源**：PDF；阅读 abstract、related work、results/conclusion、limitations。

## 今日重点趋势

1. Agent 评测正在从“大而全的昂贵环境”走向“proxy、bounded testbed、可执行 case generator”的组合。PACE、Vera、AgenticSTS 分别从成本、安全和长程记忆可控性切入。
2. Code Agent 的关键瓶颈不是只会不会写代码，而是能否用低噪声上下文、可验证中间证据和可靠协作链完成修复。ContextSniper、UA-ChatDev、Paper-replication 都把执行证据从 prompt 附属物提升为一等对象。
3. Search/RAG Agent 的下一步重点是证据状态与治理。ContextNest 关注版本和审计，Maven 关注证据编辑动作的奖励；二者都比单纯优化 retrieval relevance 更接近真实工具使用。

## 潜在研究 idea

- 构建 Agent post-training 的三层筛选器：PACE 式 proxy benchmark 做快速回归，Vera 式可执行负例做安全压力测试，ContextSniper/SkillCoach 式过程 rubric 做轨迹质量筛选。
- 将 Search Agent 轨迹表示为 evidence memory edits：query 产生候选证据，rerank 触发 add/link，fact-check 触发 drop，再用 Maven 风格奖励训练。
- 给 Code Agent 加 Context Governance：每次文件读取、测试输出、patch 依据都记录版本 hash 和 provenance，支持失败回滚和审计。
