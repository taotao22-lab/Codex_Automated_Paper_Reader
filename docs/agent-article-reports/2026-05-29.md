## General Agent

### 1. CaveAgent: Transforming LLMs into Stateful Runtime Operators
- 链接：[OpenReview 页面](https://openreview.net/forum?id=p3dlOhpqKD) | [PDF](https://openreview.net/pdf?id=p3dlOhpqKD)
- 证据来源：PDF
- 核心方法：把 agent 主工作区从文本上下文转移到持久 Python runtime。论文给出 dual-stream 架构：semantic stream 负责轻量推理，runtime stream 负责状态管理、对象注入、代码执行和跨轮持久化。
- 创新点：它不是简单的“让模型多写代码”，而是把 stateful object store、skill injection 和 verifiable runtime feedback 绑到同一执行协议里。论文明确把 runtime 可观测状态视为未来 RLVR 的反馈源，而不只是执行副产品。
- 可转化实验/模块：持久对象槽、runtime object registry、跨轮 skill injection、从执行状态直接导出 reward/verifier 信号。
- 局限：当前主验证集仍以 Tau2 / BFCL 一类任务为主，跨 GUI、跨浏览器、跨仓库的开放环境泛化还没有完全打透。
- 阅读建议：优先精读 runtime stream、Stateful Runtime Management、token efficiency 和 RLVR 相关段落。

### 2. Meta-Cognitive Memory Policy Optimization for Long-Horizon LLM Agents
- 链接：[arXiv 页面](https://arxiv.org/abs/2605.30159) | [HTML](https://arxiv.org/html/2605.30159v1)
- 证据来源：论文页面
- 核心方法：提出 Belief Entropy 作为中间记忆质量的自监督代理信号，不再只用最终成功率训练 memory policy，而是逐轮惩罚会让 latent task state 变模糊的摘要。
- 创新点：把“记忆压缩是否让 agent 失真”显式转成训练目标，这是长程 agent 比只看 outcome-based RL 更可操作的中层监督。
- 可转化实验/模块：summary-specific reward、belief probe、sub-trajectory reward、长上下文 memory policy 训练。
- 局限：Belief Entropy 仍依赖模型自身不确定性估计，外部 verifier 或真实环境反馈较弱时可能不稳。
- 阅读建议：重点看 Belief Entropy 定义、奖励拆分方式，以及 1.75M-token 场景下的扩展性论证。

### 3. SkillsInjector: Dynamic Skill Context Construction for LLM Agents
- 链接：[arXiv 页面](https://arxiv.org/abs/2605.29794) | [HTML](https://arxiv.org/html/2605.29794v1)
- 证据来源：论文页面
- 核心方法：把 skill injection 从静态检索问题改写成 context construction 问题，由 context planner 决定选哪些 skill、给多少预算，再由 set-aware renderer 决定这些 skill 如何共同呈现。
- 创新点：这篇论文的重要处不在“多一个 skill 检索器”，而在于承认 skill 描述本身就是可训练界面。agent 看到哪些工具、以什么顺序和粒度看到，都会影响最终执行。
- 可转化实验/模块：adaptive skill budgeting、execution-grounded skill preference、set-aware skill rendering。
- 局限：依赖已有 skill library；对完全未见工具和在线发现工具的情形还需要外部机制补足。
- 阅读建议：看 context planner 的训练信号和 renderer 如何利用共注入邻域信息，适合直接迁移到 MCP / tool cards 场景。

### 4. Formalizing Mathematics at Scale
- 链接：[arXiv 页面](https://arxiv.org/abs/2605.29955) | [HTML](https://arxiv.org/html/2605.29955v1)
- 证据来源：论文页面
- 核心方法：AutoformBot 用 dependency-aware task scheduling、formal verification tools、Git worktree isolation 和协作版本控制组织大规模多 agent，把教材内容转成可机器校验的 Lean 4 库。
- 创新点：它展示了“多 agent + 可执行验证 + 软件工程流程”可以在真实大型产物上闭环，而不是停在 benchmark 轨迹上。对 verifiable code agent 很有启发。
- 可转化实验/模块：DAG 任务跟踪、agent worktree 隔离、proof/check loop、版本化审查工作流。
- 局限：目标域高度结构化，且 Lean proof-checking 提供了极强 verifier；迁移到普通仓库时验证信号会更稀疏。
- 阅读建议：优先看任务调度基础设施与成功 formalization 定义，这两部分比“数学应用”本身更值得迁移。

## Search Agent

### 5. RAISE: RAG Design as an Architecture Search Problem
- 链接：[arXiv 页面](https://arxiv.org/abs/2605.30029) | [HTML](https://arxiv.org/html/2605.30029v1)
- 证据来源：论文页面
- 核心方法：把 query rewriting、chunking、retrieval depth、reranking、compression 等 RAG 设计变量统一成黑盒架构搜索问题，给出可替换 controller、统一 search space 和标准化评估层。
- 创新点：它把很多团队口中的“RAG 调参经验”升级成可复现实验对象，并明确指出不同任务上 optimizer 排名差异很大，没有统一最优策略。
- 可转化实验/模块：parameterized RAG pipeline、controller abstraction、proxy construction ablation、budgeted search。
- 局限：主要是 pipeline-level HPO 框架，对浏览器式 search agent 的交互轨迹还没有直接建模。
- 阅读建议：先看 pipeline abstraction 和 evaluation objective，再看关于 proxy / random seed / budget 的消融。

### 6. SAAS: Self-Aware Reinforcement Learning for Over-Search Mitigation in Agentic Search
- 链接：[arXiv 页面](https://arxiv.org/abs/2605.29796) | [HTML](https://arxiv.org/html/2605.29796v1)
- 证据来源：论文页面
- 核心方法：把过度搜索拆成两类错误：本不该搜却搜、证据已够却还继续搜。SAAS 用 search boundary modeling、boundary-aware reward 和 stage-wise optimization 去显式学习这条边界。
- 创新点：论文没有只惩罚“搜得多”，而是先定义 agent 什么时候真正跨过知识边界，这让 search efficiency 从启发式节流变成可训练行为。
- 可转化实验/模块：search boundary reward、search-disabled vs search-enabled rollout 对比、分阶段 RL curriculum。
- 局限：核心收益集中在 over-search 抑制；对 under-search 或复杂多源证据整合的提升还需结合其他模块。
- 阅读建议：重点看 boundary 建模与 stage-wise 训练顺序，这部分最适合迁移到 search trace RL。

### 7. Redundant or Necessary? A Benchmark for Detecting Redundant Steps in Agent Trajectories
- 链接：[arXiv 页面](https://arxiv.org/abs/2605.29893) | [HTML](https://arxiv.org/html/2605.29893v1)
- 证据来源：论文页面
- 核心方法：提出 RedundancyBench，把轨迹中每一步按对任务完成的贡献做标注，单独评估 agent 是否能识别冗余步骤，而不是只看最终答对没答对。
- 创新点：这等于把“执行效率”从附属指标提升为独立研究问题。对 search agent 尤其重要，因为过长轨迹常常来自低价值检索和重复规划。
- 可转化实验/模块：step-level redundancy labels、trajectory-level/step-level 双指标、轨迹压缩训练信号。
- 局限：当前最强方法分数仍很低，说明任务定义新但 baseline 很早期；短期更像诊断 benchmark 而非现成训练方案。
- 阅读建议：适合当作 search/code agent 轨迹 regularization 的数据接口来读，而不是只关注 benchmark 排名。

## Code Agent

### 8. MIRA: Mid-training Rubric Anchoring for Source-Aware Data Selection
- 链接：[arXiv 页面](https://arxiv.org/abs/2605.30288) | [HTML](https://arxiv.org/html/2605.30288v1)
- 证据来源：论文页面
- 核心方法：先按 source group 发现各自该看什么 rubric，再把这些 rubric 蒸馏成可扩展 student scorer，用 source-aware threshold 做大规模 mid-training 过滤。
- 创新点：关键贡献是把 rubric construction 直接纳入数据选择，而不是假设所有来源都共享一套固定标准。对 code agent 数据混合非常重要。
- 可转化实验/模块：group-wise rubric discovery、teacher-student scorer distillation、source-aware retention threshold。
- 局限：当前验证主要围绕 code-oriented mid-training；迁移到 multi-tool trace、browser log 等非文本数据需要重新定义 source group。
- 阅读建议：精读 anchor rubric 发现和 reliability-aware aggregation，这两块最适合拿来改造你自己的训练数据流水线。

### 9. Self-Trained Verification for Training- and Test-Time Self-Improvement
- 链接：[arXiv 页面](https://arxiv.org/abs/2605.30290) | [HTML](https://arxiv.org/html/2605.30290v1)
- 证据来源：论文页面
- 核心方法：利用“看不到参考答案时模型抓不住自己错误，但看到参考答案时能学会评审”的不对称性，训练 verifier 学会给可操作反馈，再把 verifier 放回 test-time refinement 和 training-time RL 循环中。
- 创新点：它把 verifier 从被动打分器提升为训练主角。对 code agent 来说，这和 execute-test-fix 闭环里的 failing test / patch critique 十分同构。
- 可转化实验/模块：self-trained verifier、verifier-in-the-loop training、feedback-quality distillation。
- 局限：论文实验集中在数学和科学推理，代码修复场景还需要真实执行反馈去补强 verifier grounding。
- 阅读建议：先看 STV 如何构造监督，再看 ViL 如何在 RL 已收敛后继续带来增益。

### 10. Cookie-Bench: Continuous On-screen Key Interaction Evaluation for Web Generation
- 链接：[arXiv 页面](https://arxiv.org/abs/2605.30000) | [HTML](https://arxiv.org/html/2605.30000v1)
- 证据来源：论文页面
- 核心方法：构建 11 domain、54 leaf、1000 query 的 WebDev benchmark，并提出 Cookie evaluator，把静态观察、agent-driven interaction、动态综合评分拆成三阶段证据链。
- 创新点：它不是参考实现比对，而是试图用“先收集屏幕证据，再给整体判断”的 evaluator 去接近人类审网站方式。这对前端 code agent 尤其有价值。
- 可转化实验/模块：reference-free web evaluator、continuous screenshot/video evidence、structured failure attribution。
- 局限：当前自动评估仍主要围绕 Web generation；更广义的软件 GUI 和跨页面流程可能需要扩展交互覆盖。
- 阅读建议：优先看 Cookie evaluator 的三阶段设计，而不是只看 leaderboard 数字。

## 今日重点趋势

1. Agent 基础设施正在从“prompt 技巧”转向“中间层协议”。CaveAgent 的 runtime、SkillsInjector 的 skill context、MIRA 的 rubric space 都是在定义 agent 与外界交互的稳定接口。
2. Search Agent 的核心问题从“能不能搜到”转向“何时该搜、搜到什么算够”。RAISE 负责结构搜索，SAAS 负责边界学习，RedundancyBench 则补上效率诊断。
3. Verifier 开始成为训练闭环的核心角色，而不是事后评分器。STV 把 verifier 放进训练内环，Cookie-Bench 把 evaluator 做成证据链，AutoformBot 用 proof checker 提供强监督。
4. Code/Agent 数据工程正在走向 source-aware 设计。MIRA 说明不同来源要有不同 rubric，这个思想同样适合代码仓库、工具调用轨迹、搜索日志和 GUI 录像。

## 潜在研究 Idea

1. 把 CaveAgent 的持久 runtime 与 MMPO 的 Belief Entropy 结合，训练能显式维护 object state 和 summary state 的 search/code agent。
2. 以 RAISE 的 pipeline abstraction 为外层、SAAS 的 boundary-aware reward 为内层，做“结构搜索 + 搜索边界 RL”联合优化的 search agent。
3. 用 RedundancyBench 的 step label 训练 code/search agent 的轨迹剪枝器，在执行前就抑制低价值工具调用。
4. 把 STV 的 verifier-in-the-loop 思想迁移到补丁生成：让 verifier 学会指出“这次修复只是 symptom reduction，不是 root-cause resolution”。
5. 用 MIRA 的 source-aware rubric 去筛代码执行轨迹、浏览器轨迹和 MCP 会话日志，而不是只按统一质量分做数据选择。
