## General Agent

### 1. CLI-Anything: Towards Agent-Native Computer Use
- 链接：[arXiv](https://arxiv.org/abs/2606.03854) | [PDF](https://arxiv.org/pdf/2606.03854)
- 证据来源：PDF
- 核心方法：论文不再把 computer use 定义为“让模型学会像人一样看屏幕和点按钮”，而是主张把现有软件包装成 `command + explicit state + deterministic feedback` 的 agent-native 协议，并用 CLI-Hub 把这种接口系统化。
- 创新点：价值不在某个单独 benchmark，而在接口边界的重新定义。作者在引言里把 `screen mechanics -> artifact semantics` 作为核心迁移，强调验证导出的工件而不是验证窗口动作本身。
- 可转化实验：可以直接为 browser/code agent 设计一组 CLI/structured-API harness，对比 GUI 轨迹训练和 agent-native 协议训练在成功率、恢复能力、状态漂移、trace 可验证性上的差异。
- 局限：这更像一个高价值范式论文和工程方向书，不是大规模量化 benchmark；“已有软件如何低成本 CLI 化”仍然需要很多后续工程。
- 阅读建议：如果你在考虑 computer-use post-training、工具接口设计或 agent benchmark 的任务定义，这篇值得放在最前面读。

### 2. When Vision Needs a Second Look: Tool-Augmented Active Perception for Earth Observation
- 链接：[OpenReview](https://openreview.net/forum?id=7yUrnyFgEq) | [PDF](https://openreview.net/pdf/a2b8ae35916447070ced1b448a4bd70350767804.pdf)
- 证据来源：PDF
- 核心方法：GeoScout-Agent 用 `GPT-5-mini + LangGraph/LangChain` 把 EO 推理变成闭环代理，允许模型在推理过程中调用代码执行、逐步 zoom、图像 sharpen、外部上下文核验，并结合 DINOv3 与 SAM3 进行区域分析。
- 创新点：不是简单在 VLM 外面包几把工具，而是把 `active perception + verification` 当成主体流程，并且明确把状态以消息列表累积，让模型能复用先前观察和工具输出。
- 关键发现：论文实验报告相对无工具基线有 17.3% 的相对提升，且收益主要来自中间失败后的修正能力，而不是单次视觉识别本身更强。
- 可转化实验：Search Agent 可以直接借鉴它的 `观察 -> 工具 -> 局部验证 -> 综合回答` 图结构，替换掉 EO 工具后就是文档/网页检索代理。
- 局限：场景是遥感，不是开放网页；同时工具上限被固定在 3 次调用，离更长轨迹代理还有距离。

### 3. WorldPack: Dynamic Frame Compression for Long-context Video World Modeling
- 链接：[OpenReview](https://openreview.net/forum?id=zJuiG3PiNJ) | [PDF](https://openreview.net/pdf/cce708ca8b3a98fe9dc8e57519799dd89a636ce2.pdf)
- 证据来源：PDF
- 核心方法：WorldPack 把长上下文记忆设计成 `trajectory packing + geometric selection` 两部分：前者在固定上下文预算下压入更多历史帧，后者根据 camera pose 和视野重叠动态分配压缩率。
- 创新点：它指出关键不是“多存几帧”，而是“根据任务相关性决定哪些历史该高保真保留”。这对 agent memory 比单纯 summary 更有启发。
- 关键发现：有效上下文从 4 帧扩到 22 帧，只增加约 16% 推理时间，并在 LoopNav 和 RECON 上稳定优于 Oasis、Mineworld、DIAMOND、NWM。
- 可转化实验：可以把同样的预算分配逻辑迁到 browser trajectory、GUI screenshot history、repo navigation memory 里，而不是把所有旧上下文均匀压缩。
- 局限：论文对象仍是视频 world model，不是文本代理；迁移时需要重新定义“空间相关性”在工具使用里的等价物。

### 4. Cosmos 3: Omnimodal World Models for Physical AI
- 链接：[arXiv](https://arxiv.org/abs/2606.02800) | [PDF](https://arxiv.org/pdf/2606.02800)
- 证据来源：PDF
- 核心方法：Cosmos 3 试图用统一的 `Mixture-of-Transformers` 主干同时处理语言、图像、视频、音频和动作 token，把理解、生成、模拟和 policy 建模放到一个 omnimodal 框架内。
- 创新点：把 action 明确视作第一类模态，而不是附属控制信号；这使“物理世界动作”和“语言/视频推理”共享表示空间。
- 可转化实验：如果你关心 General Agent 训练数据构造，可以重点看它如何组织多模态输入输出模式、如何处理不同模态的编码和生成路径，以及其开放数据/评测资产释放策略。
- 局限：这是大规模系统技术报告，强在平台和覆盖面，弱在对单个 agent 训练问题给出细致可控的因果拆解。

### 5. OmniHalluc-L: Counterfactual Benchmarking and Modality-Perturbation Reliability Calibration for Long-Form Omni Hallucination
- 链接：[arXiv](https://arxiv.org/abs/2606.03614) | [PDF](https://arxiv.org/pdf/2606.03614)
- 证据来源：PDF
- 核心方法：论文把长视频 Omni assistant 的幻觉重写为“真实证据的错误绑定”，用支持/反事实成对 claims 做 strict-pair accuracy，并提出 MPRC 用音频负探针校准支持置信度。
- 创新点：它评测的不是“有没有凭空编造”，而是“证据在，但绑定错了 speaker / moment / scene / modality”。这和 Search Agent 的引用错配、证据错链问题高度同构。
- 关键发现：开源 Omni 模型在 pair-level binding 上显著落后，MPRC 在不改 backbone 的前提下仍能带来稳定增益。
- 可转化实验：可把同样的 counterfactual reliability probing 搬到网页引用验证、多跳检索、工具调用依据核验。
- 局限：当前评测形式还是单 claim 校验，不覆盖完整多轮交互。

## Search Agent

### 6. Skill Is Not Document: A Query-Conditional Benchmark and Two-Stage Retriever for LLM Agent Skill Routing
- 链接：[arXiv](https://arxiv.org/abs/2606.03565) | [PDF](https://arxiv.org/pdf/2606.03565)
- 证据来源：PDF
- 核心方法：作者把 skill retrieval 和 document retrieval 区分开来，强调 top-K 不是看单个技能是否相关，而是看一组技能在该 query 下能否协同完成任务。为此构建了 R3-Skill，并把 LLM 拒绝样本变成显式的 compatibility 负监督。
- 创新点：`Reject-as-Resource` 这点很重要。常见数据合成流程会把 rejected skill sets 丢弃，这篇则把它转化为 `哪些技能不该一起检索` 的训练信号，并把该信号主要放到 cross-encoder reranker 阶段。
- 关键发现：两阶段系统在 R3-Skill 上拿到 Hit@1 0.7714、NDCG@10 0.8327、Set-Compat 0.3525，且 compatibility 收益主要体现在 reranker，而非 bi-encoder。
- 可转化实验：对于 Codex/Claude/Gemini 这类按需加载 skills 的系统，这篇几乎可以直接指导 skill router 数据构造、hard negative 设计和评测指标。
- 局限：数据虽接近真实请求，但目前仍是 skill routing 子问题，不等于完整 agent execution 成功率。

### 7. Can LLM Rerankers Predict Their Own Ranking Performance?
- 链接：[arXiv](https://arxiv.org/abs/2606.03535) | [PDF](https://arxiv.org/pdf/2606.03535)
- 证据来源：PDF
- 核心方法：研究 `reranker-internal QPP`，即 reranker 能否对自己刚产出的排序质量做内部估计。作者比较 self-consistency、多次采样一致性和 verbalized confidence，并提出 Verb-Num / Verb-List 做轻量监督校准。
- 创新点：把 query performance prediction 从“外挂一个外部 predictor”改成“让 reranker 自我报告不确定性”，这更适合 Agent 在检索链路中做动态分支控制。
- 关键发现：training-free 的 self-consistency 已经能接近或对标现有无监督 SOTA，而直接 verbalized confidence 明显过度自信；轻量监督后置信表达才变得更可用。
- 可转化实验：Search Agent 可用这一信号做 `是否继续检索`、`是否需要换查询`、`是否触发 citation verification` 的路由开关。
- 局限：实验仍是 TREC 风格 reranking，不是完整网页搜索代理；多级相关性和长程 reasoning 的联动还没展开。

## Code Agent

### 8. Synthetic Hallucinations, Real Gains: Hard Negatives from Frontier Models for FIM Hallucination Mitigation
- 链接：[OpenReview](https://openreview.net/forum?id=OoYmjSlQap) | [PDF](https://openreview.net/pdf/604729aa6d5d0c509db4971e05ea7bede25aebe1.pdf)
- 证据来源：PDF
- 核心方法：针对 IDE 中最常见的 FIM 幻觉，作者不用执行沙箱，也不用大规模人工偏好数据，而是让 frontier code models 生成“看起来合理但项目里并不存在”的 hard negatives，再把它们与真实开发者编辑配对做 SFT。
- 创新点：把 `frontier models 作为错误生成器`，专门为小模型蒸出高难负例，这非常适合 code agent / code completion 的后训练数据构造。
- 关键发现：论文摘要报告在 Delulu 上，Qwen2.5-Coder-7B-Instruct 的 exact match 提升 18.8 点，3B 版本也有明显收益；同时作者做了 size/type/language/difficulty-aware fool rate 等多轴消融。
- 可转化实验：可以把这套 recipe 扩展到 repo-level patching，负例不再只是 FIM 补全，也可以是错误 API 调用、错误 import、错误 tool call、错误 test fix。
- 局限：目标仍是 mid-keystroke autocomplete，不是多文件 issue resolution；但数据构造思路极强。

### 9. From Rocq to Metal: A Pipeline for Formally Verified Microcontroller Firmware
- 链接：[arXiv](https://arxiv.org/abs/2606.02651) | [PDF](https://arxiv.org/pdf/2606.02651)
- 证据来源：PDF
- 核心方法：作者构建 Encore!，让 Rocq 提取出的 Scheme 能在 Cortex-M 裸机上跑起来，并把 firmware 核心设计成纯状态转移函数，从而把业务逻辑完全放进可证明区域。
- 创新点：最有价值的是把“AI 生成代码如何验”落到架构层：不是事后 review，而是让 theorem signatures 和 proof obligations 成为主要接口。
- 关键发现：论文展示了 Rocq-proven firmware 跑在 Cortex-M35P / Ledger Flex 这条链路，且把未验证 TCB 固定在一个不随业务复杂度增长的边界内。
- 可转化实验：Code Agent 可以借鉴“把问题重构成可验证状态转移核心”的思路，在生成代码时同步产出更强的规格与检查接口。
- 局限：应用域是嵌入式固件，工程假设强；对通用仓库修复的直接迁移需要额外抽象。

### 10. Lean-GAP: A Dataset of Formalized Graduate Algebra Problems
- 链接：[arXiv](https://arxiv.org/abs/2606.02588) | [PDF](https://arxiv.org/pdf/2606.02588)
- 证据来源：PDF
- 核心方法：构造 430 道研究生代数题的 Lean 4 正式化数据，并把 `PDF-to-LaTeX -> autoformalization -> informal-formal correspondence verification` 做成一条系统流程。
- 创新点：论文明确指出 verification 才是最微妙、最费人的步骤，并提供 suspicious-statement patterns 与语义检查来阻止“形式上能 elaborates 但数学上空洞”的结果。
- 关键发现：作者报告了不同 autoformalization 模型的对比，文中结果显示 GPT-5 在单次系统里领先，但整体成功率仍远没到“自动化完成”的程度。
- 可转化实验：这对 Code Agent / theorem-proving agent 的启发在于，训练数据不能只存成功证明，还要保留失败模式、空洞形式化模式和验证检查器。
- 局限：任务域是数学形式化，不是软件仓库；但数据构造与 verifier 设计很有迁移价值。

## 今日重点趋势

1. 本批次最有价值的论文都在强调“接口和反馈要为代理而设计”。`CLI-Anything` 重写软件接口，`Skill Is Not Document` 重写 skill retrieval 监督，`Synthetic Hallucinations` 重写 code-model 负例构造，`OmniHalluc-L` 重写可靠性评测。
2. `verification-aware agenting` 是今天的第二主线。`GeoScout-Agent` 把验证放进工具链，`Rocq to Metal` 把验证放进架构，`Lean-GAP` 把验证放进数据构造，`Reranker QPP` 则把不确定性估计放回排序器内部。
3. 长上下文系统设计继续变强。`WorldPack` 说明真正关键的是按相关性分配记忆预算，而不是盲目扩窗；这和通用 agent 的 memory compression 问题高度一致。

## 潜在研究 Idea

1. 做一个 `agent-native interface benchmark`：同一任务分别暴露 GUI、CLI、JSON state、artifact verifier 四种接口，比较训练和推理效果。
2. 用 `reject-as-resource` 思路构造 skill/router 训练集，把“错误工具组合”“错误 citation 组合”“错误子任务组合”都转成显式负监督。
3. 用 `Synthetic Hallucinations` 的 recipe 生成 repo-level hard negatives，让 code agent 学会避免局部合理但全局不存在的 API / symbol / patch。
4. 把 `WorldPack` 的压缩策略迁到浏览器轨迹、代码导航轨迹和多轮检索记忆，比较 uniform summary 与 relevance-aware compression。
