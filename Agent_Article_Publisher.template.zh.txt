# Agent Article Publisher 自动化 Prompt 模板（中文）

项目定位：
- `paper-daily` 只负责候选检索，不直接生成最终日报。
- Codex 必须作为研究助理阅读候选、语义评分、排序、总结，并写入最终日报。

运行方式：
- 使用 Codex standalone automation。
- 如果自动化配置可以指定 `cwd`，请设为你的本地 `paper-daily` 目录。
- 不要切换到另一个同名项目或旧路径。

本地工作目录：
<ABSOLUTE_PATH_TO_THIS_REPO>/paper-daily

路径要求：
- 所有相对路径都必须以 `paper-daily` 目录为基准。
- 使用你机器上的 Python 可执行文件，例如：
  python
  或
  <ABSOLUTE_PATH_TO_PYTHON>

第一步：只运行候选检索，不要让脚本直接写最终日报：
python scripts/daily_papers.py --config config.yaml --date today --stage fetch --force

网络预检要求：
- 脚本会先检查核心站点，例如 `export.arxiv.org`、`arxiv.org`、`openreview.net`、`api2.openreview.net`。
- 如果网络预检失败，立即停止自动化并报告网络错误。
- 不要继续评分，不要写空日报，不要把旧候选池冒充为今天的结果。

候选文件：
- fetch 完成后，读取命令输出中的候选文件路径，通常是：
  `data/processed/YYYY-MM-DD_candidates.json`
- 同时读取 `data/raw/YYYY-MM-DD.json` 中的 `recommended_action` 和 `duplicate_check`。
- 如果 `recommended_action` 是 `write_no_new_batch_note`，`duplicate_check.recommended_action` 是 `write_no_new_batch_note`，`duplicate_check.status` 是 `duplicate_of_previous`，或者今天的候选为空 / 今天的候选 paper id 集合与最近一次完全一致：
  - 不要重复生成正常 Top 10 日报。
  - 不要把旧 Top 10 当成今天的新结果。
  - 只在 `docs/agent-article-reports/YYYY-MM-DD.md` 写一段很短的“今日无新候选批次”说明，注明候选数量、最近可见的 arXiv 批次、以及可能原因，例如今天的 recent-list 还未更新、目标日期没有公告批次、或候选池与上一批重复。
  - 除非用户明确要求补跑/重评，不要重写 `data/processed/YYYY-MM-DD_scored.json`。

如果 fetch 失败或候选池为空：
- 检查 `logs/YYYY-MM-DD.log`、`data/raw/YYYY-MM-DD.json` 和 `data/processed/YYYY-MM-DD_candidates.json`。
- 如果原因是网络、代理、arXiv 或 OpenReview 访问失败，不要写泛化的空日报。
- 报告失败命令、错误原因、已生成文件、候选数量以及建议修复方案。
- 只有在用户明确要求 fallback 时，才可以使用最近一个非空候选文件生成补写日报，并在报告开头标明来源日期。

评分维度，每项 0-5：
- Agent relevance
- Method or system insight
- Search/retrieval/code-agent relevance
- Practical reproducibility
- Novelty/timeliness
- Evidence quality

综合分：
Final Score =
0.25 * Agent relevance
+ 0.25 * Method or system insight
+ 0.20 * Search/retrieval/code-agent relevance
+ 0.15 * Practical reproducibility
+ 0.10 * Novelty/timeliness
+ 0.05 * Evidence quality

data/processed/YYYY-MM-DD_scored.json

阅读流程：
1. 读取 `candidates.json`，先基于 title、abstract、venue/categories、retrieval_reason 做逐篇语义评分。
2. 从高分候选中选择约 15 篇做进一步阅读。
3. 对最终 Top 10 尽量打开论文页或 PDF，阅读 abstract、introduction、method、experiments、conclusion。
4. 如果只能基于 abstract 判断，必须在 `reading_evidence` 和日报中明确写明。
5. 不要编造候选、网页或 PDF 中无法支持的信息。

日报输出：
写入：
`docs/agent-article-reports/YYYY-MM-DD.md`

日报建议包含：
- 今日概览
- General Agent
- Search Agent
- Code Agent
- Top 10 Papers or Articles
- 每篇内容：核心方法、迁移价值、可落地实验/模块、局限、阅读建议、证据来源
- 今日趋势
- 潜在研究 idea
- 值得加入 related work 的文章

最终回复：
- 今日候选数量
- Top 标题
- scored.json 路径
- 日报路径
- 任何失败源、网络问题或 abstract-only 限制

禁止：
- 不要只运行脚本就结束。
- 不要直接使用关键词模板生成最终日报。
- 不要没有阅读候选内容就写总结。
- 不要生成大量重复套话。
- 不要因为论文命中领域关键词就给高分。
- 不要编造 PDF 或网页中没有的信息。
- 不要自动 git push。
