# Agent Article Publisher 自动化 Prompt 模板（中文）

项目定位：
- `paper-daily` 只是候选检索脚本工具，不是日报生成器。
- Codex 必须作为科研助理阅读候选、语义评分、排序、总结，并写入最终日报。

运行方式：
- 使用 Codex standalone automation。
- 如果自动化配置可以指定 `cwd`，请设为你的本地 `paper-daily` 目录。
- 不要切换到其他同名或旧路径项目。

本地路径：
C:\Users\24644\Desktop\Codex_Automated_Paper_Reader\paper-daily

路径要求：
- 所有相对路径都必须以 `paper-daily` 目录为基准。
- 使用当前环境中的 Python 可执行文件：
  python
- 仓库根目录是：
  C:\Users\24644\Desktop\Codex_Automated_Paper_Reader
- 日报提交目录是：
  C:\Users\24644\Desktop\Codex_Automated_Paper_Reader\docs\agent-article-reports

第一步：只运行候选检索，不要让脚本生成日报。自动化固定检索 3 天前的候选，避免当天源站批次尚未更新：
python scripts/daily_papers.py --config config.yaml --date 3-days-ago --stage fetch --force

日期规则：
- `--date 3-days-ago` 会解析为运行当天往前 3 天的目标日期。
- 后续所有 `YYYY-MM-DD` 都必须使用这个目标日期，而不是自动化运行当天。
- 例如在 2026-05-31 运行时，目标日期是 2026-05-28，应读取 `data/processed/2026-05-28_candidates.json`，写入 `docs/agent-article-reports/2026-05-28.md`，提交信息使用 `文档：新增 2026-05-28 Agent 文章日报`。

网络预检要求：
- 脚本会先检测 `export.arxiv.org`、`arxiv.org`、`openreview.net`、`api2.openreview.net` 等核心网址。
- 如果网络预检失败，立即停止自动化并报告网络错误。
- 不要继续评分，不要写空日报，不要用旧候选池冒充当天结果。

候选文件：
- fetch 完成后读取命令输出中的候选文件路径，通常是：
  data/processed/YYYY-MM-DD_candidates.json
- 同时读取 `data/raw/YYYY-MM-DD.json` 中的 `recommended_action` 和 `duplicate_check`。
- 这里的 `YYYY-MM-DD` 是 `3-days-ago` 解析出的目标日期。
- 如果 `recommended_action` 是 `write_no_new_batch_note`、`duplicate_check.recommended_action` 是 `write_no_new_batch_note`、`duplicate_check.status` 是 `duplicate_of_previous`，或当天 candidates 为空 / paper id 集合与最近一天 candidates 完全一致：
  - 不要重复生成正常 Top 10 日报。
  - 不要沿用旧 Top 10 冒充当天新推荐。
  - 写一个很短的“今日无新候选批次”说明到 `docs/agent-article-reports/YYYY-MM-DD.md`，注明候选数量、最近可见 arXiv 批次、主要原因（例如 arXiv 今日 recent-list 尚未更新、当天无公告批次、或候选池与上一批重复）。
  - 除非用户明确要求补跑/重评，不要重写 `data/processed/YYYY-MM-DD_scored.json`。

如果 fetch 失败或候选池为空：
- 检查 `logs/YYYY-MM-DD.log`、`data/raw/YYYY-MM-DD.json` 和 `data/processed/YYYY-MM-DD_candidates.json`。
- 如果原因是网络、代理、arXiv 或 OpenReview 访问失败，不要写泛化空日报。
- 报告失败命令、错误原因、已生成文件、候选数量和建议修复方案。
- 只有在用户明确要求 fallback 时，才可使用最近一个非空 candidates 文件生成补跑日报，并在报告开头标明来源日期。

研究背景：
我关注 General Agent、Search Agent 和 Code Agent 方向。参考 DeepSeek-V3.2 技术报告，我更关心“高效长上下文 + 可扩展强化学习 + 大规模 agentic task synthesis”如何把强推理能力稳定迁移到真实工具使用场景：模型不仅要会解题，还要能在复杂交互环境中规划、调用工具、检索信息、读写代码、验证结果并持续修正。希望日报优先筛选能启发 Agent 训练数据构造、post-training/RL、tool-use reasoning、长上下文系统设计、代码执行闭环和工程化部署的论文或技术文章。

推荐原则：
- 优先推荐能迁移到 General Agent、Search Agent 和 Code Agent 研究问题中的通用 AI 方法、系统、训练流程和评测论文/文章。
- 不要只因为论文包含领域关键词就给高分。
- 相邻领域论文只要方法可迁移，也可以高分。
- 对直接领域论文也要看方法贡献、实验质量、可复现性、数据构造方式和可复用工程价值。

重点关注：
- General Agent：reasoning-in-tool-use、长程规划、任务分解、工具调用协议、记忆与上下文压缩、反思/自修正、多轮交互鲁棒性、Computer-use/GUI/Browser Agent、可扩展 agentic post-training。
- Search Agent：搜索任务合成、查询规划、网页浏览、RAG、reranking、信息寻源、事实核验、答案验证、引用可信度、长上下文检索整合，以及搜索轨迹如何作为强化学习或监督微调数据。
- Code Agent：代码生成与编辑、仓库级理解、issue resolution、测试生成、自动调试、执行反馈、错误定位、代码审查、SWE-bench 类评测，以及“思考-工具调用-运行测试-修复”的闭环训练。
- Agent Training and Evaluation：大规模 agentic task synthesis、复杂环境构造、RL/post-training compute scaling、trajectory-level evaluation、instruction-following robustness、tool-use benchmark、沙箱和权限控制。

负向过滤：
- 与 Agent 架构、搜索/检索、代码任务或评测无关的纯应用论文。
- 纯医学、材料、金融、推荐系统、教育 tutoring、交通预测、蛋白质或机器人控制论文，除非方法可以直接迁移到 Agent。
- 只有关键词命中但没有 Agent 方法、训练数据、工具链、评测环境、执行反馈或工程启发的文章。

评分维度，每项 0-5：
- Methodological Relevance
- Inspiration Value
- Transferability to My Research
- Paper Quality
- Novelty / Timeliness
- Actionability

综合分：
Final Score =
0.25 * Methodological Relevance
+ 0.25 * Inspiration Value
+ 0.20 * Transferability to My Research
+ 0.15 * Paper Quality
+ 0.10 * Novelty / Timeliness
+ 0.05 * Actionability

评分输出：
保存到：
data/processed/YYYY-MM-DD_scored.json

注意：这里的 `YYYY-MM-DD` 必须是目标日期，即 3 天前日期。

阅读流程：
1. 读取 candidates.json，先基于 title、abstract、venue/categories、retrieval_reason 做逐篇语义评分。
2. 从高分候选中选择约 15 篇进一步阅读。
3. 对最终 Top 10 尽量打开论文页面或 PDF，阅读 abstract、introduction、method、experiments、conclusion。
4. 如果只能基于 abstract 判断，必须在 `reading_evidence` 和日报中明确写明。
5. 不要编造候选、网页或 PDF 中无法支持的信息。

日报输出：
写入：
docs/agent-article-reports/YYYY-MM-DD.md

注意：日报日期必须是目标日期，即 3 天前日期。

日报建议包含：
- 今日概览
- General Agent
- Search Agent
- Code Agent
- Top 10 Papers or Articles
- 每篇论文/文章的核心方法、迁移价值、可转化实验/模块、局限、阅读建议、证据来源
- 今日重点趋势
- 潜在研究 idea
- 值得加入 related work 的论文/文章

GitHub 提交要求：
- 自动化应每天 7:00 运行，并提交 3 天前日期对应的日报。
- 只提交 `docs/agent-article-reports` 中本次新增或修改的日报/索引文件。
- 不要提交 `paper-daily/data/`、`paper-daily/logs/`、虚拟环境或临时文件。
- 如果有可提交变化，执行：
  git add docs/agent-article-reports
  git commit -m "文档：新增 YYYY-MM-DD Agent 文章日报"
  git push origin main

最后回复用户：
- 目标日期
- 今日候选数量
- Top 10 标题
- scored.json 路径
- 日报路径
- 任何失败源、网络/代理问题或 abstract-only 限制

禁止：
- 禁止只调用脚本就结束。
- 禁止直接使用关键词模板生成最终报告。
- 禁止没阅读候选内容就总结。
- 禁止生成大量重复套话。
- 禁止因为论文命中领域关键词就给高分。
- 禁止编造 PDF 或正文中没有的信息。
- 禁止提交候选数据、日志、虚拟环境或临时文件。
