# Paper Reader 自动化 Prompt 模板（中文）

项目定位：
- `paper-daily` 只是候选检索脚本工具，不是日报生成器。
- Codex 必须作为科研助理阅读候选、语义评分、排序、总结，并写入最终日报。

运行方式：
- 使用 Codex standalone automation。
- 如果自动化配置可以指定 `cwd`，请设为你的本地 `paper-daily` 目录。
- 不要切换到其他同名或旧路径项目。

本地路径：
<ABSOLUTE_PATH_TO_THIS_REPO>/paper-daily

路径要求：
- 所有相对路径都必须以 `paper-daily` 目录为基准。
- 使用你的 Python 可执行文件，例如：
  python
  或
  <ABSOLUTE_PATH_TO_PYTHON>

第一步：只运行候选检索，不要让脚本生成日报：
python scripts/daily_papers.py --config config.yaml --date today --stage fetch --force

网络预检要求：
- 脚本会先检测 `export.arxiv.org`、`arxiv.org`、`openreview.net`、`api2.openreview.net` 等核心网址。
- 如果网络预检失败，立即停止自动化并报告网络错误。
- 不要继续评分，不要写空日报，不要用旧候选池冒充当天结果。

候选文件：
- fetch 完成后读取命令输出中的候选文件路径，通常是：
  data/processed/YYYY-MM-DD_candidates.json
- 同时读取 `data/raw/YYYY-MM-DD.json` 中的 `recommended_action` 和 `duplicate_check`。
- 如果 `recommended_action` 是 `write_no_new_batch_note`、`duplicate_check.recommended_action` 是 `write_no_new_batch_note`、`duplicate_check.status` 是 `duplicate_of_previous`，或当天 candidates 为空 / paper id 集合与最近一天 candidates 完全一致：
  - 不要重复生成正常 Top 10 日报。
  - 不要沿用旧 Top 10 冒充当天新推荐。
  - 写一个很短的“今日无新候选批次”说明到 `reports/YYYY-MM-DD.md`，注明候选数量、最近可见 arXiv 批次、主要原因（例如 arXiv 今日 recent-list 尚未更新、当天无公告批次、或候选池与上一批重复）。
  - 除非用户明确要求补跑/重评，不要重写 `data/processed/YYYY-MM-DD_scored.json`。

如果 fetch 失败或候选池为空：
- 检查 `logs/YYYY-MM-DD.log`、`data/raw/YYYY-MM-DD.json` 和 `data/processed/YYYY-MM-DD_candidates.json`。
- 如果原因是网络、代理、arXiv 或 OpenReview 访问失败，不要写泛化空日报。
- 报告失败命令、错误原因、已生成文件、候选数量和建议修复方案。
- 只有在用户明确要求 fallback 时，才可使用最近一个非空 candidates 文件生成补跑日报，并在报告开头标明来源日期。

评分维度，每项 0-5：
- Methodological Relevance
- Inspiration Value
- Transferability to My Research
- Paper Quality
- Novelty / Timeliness
- Actionability

综合分：
Final Score =
0.25 * Methodological Relevance
+ 0.25 * Inspiration Value
+ 0.20 * Transferability to My Research
+ 0.15 * Paper Quality
+ 0.10 * Novelty / Timeliness
+ 0.05 * Actionability

评分输出：
保存到：
data/processed/YYYY-MM-DD_scored.json

阅读流程：
1. 读取 candidates.json，先基于 title、abstract、venue/categories、retrieval_reason 做逐篇语义评分。
2. 从高分候选中选择约 15 篇进一步阅读。
3. 对最终 Top 10 尽量打开论文页面或 PDF，阅读 abstract、introduction、method、experiments、conclusion。
4. 如果只能基于 abstract 判断，必须在 `reading_evidence` 和日报中明确写明。
5. 不要编造候选、网页或 PDF 中无法支持的信息。

日报输出：
写入：
reports/YYYY-MM-DD.md

日报建议包含：
- 今日概览
- Top 10 Papers
- 每篇论文的核心方法、迁移价值、可转化实验/模块、局限、阅读建议、证据来源
- 今日重点趋势
- 潜在研究 idea
- 值得加入 related work 的论文

最后回复用户：
- 今日候选数量
- Top 10 标题
- scored.json 路径
- 日报路径
- 任何失败源、网络/代理问题或 abstract-only 限制

禁止：
- 禁止只调用脚本就结束。
- 禁止直接使用关键词模板生成最终报告。
- 禁止没阅读候选内容就总结。
- 禁止生成大量重复套话。
- 禁止因为论文命中领域关键词就给高分。
- 禁止编造 PDF 或正文中没有的信息。
- 禁止自动 git push。
